def _apply_fix(
    name: str, fix_fn,
    severity: Literal["safe_alias", "semantic_guess", "destructive"] = "safe_alias",
    confidence: float = 1.0,
):
    # 强制 category 检查
    allowed = DEFAULT_ALLOWED_AUTOFIX_CATEGORIES  # {SYNTACTIC_ALIAS, SCHEMA_DEFAULT, CONTEXT_SAFE}
    severity_to_category = {
        "safe_alias": AutoFixCategory.CONTEXT_SAFE,
        "semantic_guess": AutoFixCategory.SEMANTIC_GUESS,
        "destructive": AutoFixCategory.DESTRUCTIVE,
    }
    cat = severity_to_category.get(severity, AutoFixCategory.SEMANTIC_GUESS)
    if cat not in allowed:
        report.entries.append(AutoFixEntry(
            rule_id=name, path="/", old_value="<skipped>", new_value="<skipped>",
            severity=severity, confidence=confidence,
            message=f"Fix '{name}' skipped: category '{severity}' not in allowed categories",

### Phase B：P1 preflight 通用校验增强（3 项，依赖 Phase A 的 fail-closed 原则）

#### B1. 新增 A007：多 annular_groove z 区间冲突检测

**文件：** `dialects/axisymmetric/dialect.py`
**位置：** `preflight_component` 函数，lines 300 之前（A006 之后）
**修复方案：**
```python
# A007: 多 annular_groove z 区间冲突检测
groove_nodes = [n for n in nodes if n.op == "cut_annular_groove"]
groove_intervals: list[tuple[str, float, float]] = []  # (node_id, z_start, z_end)

for g in groove_nodes:
    p = g.params if not g.typed_params else g.typed_params
    inner = float(p.get("inner_dia_mm", 0))
    outer = float(p.get("outer_dia_mm", 0))
    depth = float(p.get("depth_mm", 0))
    side = p.get("side", "front")
    z_position = p.get("z_position_mm")

    # A007a: 缺少 z_position_mm 时警告（不阻塞，但提示风险）
    if z_position is None:
        issues.append(ValidationIssue(
            stage=stage,
            code="a007_groove_z_position_missing",
            message=(
                f"cut_annular_groove '{g.id}' 缺少 z_position_mm，将默认使用端面位置。"
                f"多个 groove 同时缺失 z_position_mm 会导致 z 区间重叠。"
            ),
            severity="warning",
            node_id=g.id,
        ))
    else:
        z_pos = float(z_position)
        z_start = z_pos
        z_end = z_pos + depth if side == "front" else z_pos - depth
        groove_intervals.append((g.id, z_start, z_end))

# A007b: 检测区间重叠
for i, (id1, s1, e1) in enumerate(groove_intervals):
    for id2, s2, e2 in groove_intervals[i + 1:]:
        if s1 < e2 and s2 < e1:
            issues.append(ValidationIssue(
                stage=stage,
                code="a007_groove_z_overlap",
                message=(
                    f"cut_annular_groove '{id1}' (z={s1:.1f}→{e1:.1f}) 与 "
                    f"'{id2}' (z={s2:.1f}→{e2:.1f}) 的 z 区间重叠。"
                    f"请为每个 groove 指定不重叠的 z_position_mm。"
                ),
                severity="error",
                node_id=id1,
                expected={"non_overlapping_z_intervals": True},
                actual={"groove_1": [s1, e1], "groove_2": [s2, e2]},
            ))
```
**正向性：** 对所有使用多 groove 的零件都正向（法兰多道密封槽、轴多道油槽等）。
**风险：** 低。新增规则，不修改现有逻辑。

#### B3. 新增 A009：revolve_profile 最小 r 合理性校验

**文件：** `dialects/axisymmetric/dialect.py`
**位置：** `preflight_component` 函数，A001 之后（约 lines 144 之后）
**修复方案：**
```python
# A009: revolve_profile 最小 r 合理性校验
# 如果有 cut_center_bore，revolve_profile 的最小 r 应该 >= bore 半径
# 如果无 cut_center_bore，revolve_profile 的最小 r 应该 >= 5mm（避免伪中心细颈）
for n in revolve_nodes:
    stations = n.params.get("profile_stations", [])
    if not stations:
        continue
    min_r = min(float(s.get("r_mm", 0)) for s in stations)

    if center_bore_radius > 0:
        # 有 cut_center_bore：revolve_profile 不应处理中心区域
        if min_r < center_bore_radius:
            issues.append(ValidationIssue(
                stage=stage,
                code="a009_revolve_r_below_bore",
                message=(
                    f"revolve_profile '{n.id}' 最小 r_mm={min_r} 小于中心孔半径 {center_bore_radius}。"
                    f"中心孔由 cut_center_bore 处理，revolve_profile 只描述外轮廓，"
                    f"最小 r 应 >= 轮毂外圆半径。"
                ),
                severity="warning",
                node_id=n.id,
                expected={"min_r_gte": center_bore_radius},
                actual={"min_r": min_r},
            ))
    else:
        # 无 cut_center_bore：避免伪中心细颈
        if min_r < 5.0:
            issues.append(ValidationIssue(
                stage=stage,
                code="a009_revolve_r_too_small",
                message=(
                    f"revolve_profile '{n.id}' 最小 r_mm={min_r} 过小，可能产生伪中心细颈。"
                    f"若无中心孔，最小 r 应 >= 5mm；若需中心孔，请使用 cut_center_bore。"
                ),
                severity="warning",
                node_id=n.id,
                expected={"min_r_gte": 5.0},
                actual={"min_r": min_r},
            ))
```
**正向性：** 对所有使用 revolve_profile 的零件都正向。
**风险：** 低。warning 级别，不阻塞生成，但提示风险。

#### C1. NODE_PARAMS_SYSTEM_PROMPT 增加"IR 能力边界"与"工程布局规则"章节

**文件：** `authoring/prompt_builders.py`
**位置：** lines 86-144（NODE_PARAMS_SYSTEM_PROMPT 定义）
**修复方案：** 在现有 prompt 开头增加结构化章节：
```python
# 在 NODE_PARAMS_SYSTEM_PROMPT 的开头（line 86 之后）增加：

## IR 能力边界（axisymmetric 方言 v0.2.0）

### 能表达的 op（7 个）
1. revolve_profile: 分段线性外轮廓旋转生成主体（只描述外轮廓，中心孔由 cut_center_bore 处理）
2. cut_center_bore: 中心孔
3. cut_annular_groove: 环形凹槽（必须指定 z_position_mm，多道 groove 的 z 区间不能重叠）
4. cut_circular_hole_pattern: 圆孔阵列（PCD + hole_dia/2 必须在 body 最大半径内）
5. cut_rim_slot_pattern: 简化对称轮缘槽（非真实枞树形，half_width 应交替宽/窄）
6. apply_safe_chamfer: 选择性倒角（target 应明确，避免 "all_external_edges" 破坏拓扑）
7. apply_safe_fillet: 选择性圆角（用于截面过渡）

### 无法表达的特征（不要尝试生成）
- 真实枞树形榫槽（多齿、非对称、齿根圆角）— 当前 slot_profile 只支持对称闭合多边形
- 篦齿密封（尖齿迷宫密封）
- 冷却孔阵列（带角度的冷却空气孔）
- 沉孔/锪平（螺栓头容纳）
- 等强度曲线轮廓（双曲线/幂函数）— 当前 profile_stations 只支持分段线性
- 锁片槽（防叶片窜动）

### 工程布局规则（必须遵守）
1. revolve_profile 的最小 r 必须是轮毂外圆（如 130mm），不能是 1 或接近 0
   - 中心孔由 cut_center_bore 单独处理
   - revolve_profile 只描述外轮廓
2. cut_circular_hole_pattern 的 PCD 必须使 PCD/2 + hole_dia/2 < body 最大半径
   - 螺栓孔 PCD/2 应小于轮毂外圆半径
   - 孔不能位于中心孔区域（PCD/2 - hole_dia/2 > 中心孔半径）
3. cut_annular_groove 必须指定 z_position_mm
   - 多道 groove 的 z 区间不能重叠
   - z_position_mm 应基于 body 的实际 z 范围
4. cut_rim_slot_pattern 使用"简化对称轮廓"
   - stations 的 half_width 应交替宽/窄（lobe/neck 模式）
   - 不要尝试生成真实枞树形（IR 无法表达非对称齿形）
5. apply_safe_chamfer 的 target 应明确
   - 避免使用 "all_external_edges"（会破坏复杂几何拓扑）
6. 截面突变处应使用 apply_safe_fillet 而非 chamfer
   - 圆角半径 R3-R10
```
**正向性：** 对所有零件都正向，规则通用。
**风险：** 低。仅增加 prompt 内容，不修改代码逻辑。

3. **不引入新 op**：这是 P4 范畴。当前修复聚焦于"让现有 IR 正确工作"。
4. **B2 优先确认 A003 覆盖度**：若 A003 已覆盖 PCD 校验，则 B2 改为"增强错误信息"而非"新增规则"。
