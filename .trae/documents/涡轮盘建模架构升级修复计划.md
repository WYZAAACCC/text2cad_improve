# 涡轮盘建模架构升级修复计划

## Summary

基于对 `turbine_disc` 测试材料的深度诊断和对 `generative_cad` 子系统的架构审计，识别出 **16 个问题**分布在 **4 个根因层**。本计划采用"架构核心全修复 + 保持 IR 单值语义"策略，通过 **7 个修复批次**解决根因，使涡轮盘正确生成且同类问题不再复发。

**核心诊断结论**：涡轮盘建模失败不是单一 bug，而是 **Prompt 通道断裂 + Schema 约束不足 + Handler 算法缺陷** 三层叠加的结果。报告中识别的 4 个表面问题（Vite 端口、Slot 深度递减、Bore 共面切割、Chamfer 降级）都是这三层根因的表征。

**策略选择**（用户已确认）：
- 修复范围：架构核心全修复（Prompt + Schema + Handler + 配置）
- revolve_profile：保持单值语义 + 检测报错 + 引导重试（不改 IR 语义）

---

## Current State Analysis

### 根因层 1：Prompt 通道断裂（最严重）

生产 pipeline (`authoring/pipeline.py`) 三处 LLM 调用 **完全不发送 system prompt**：

```python
# pipeline.py 第 215, 267, 295-296 行
messages=[{"role": "user", "content": user_request}]  # 只有 user，无 system
```

而 `authoring/prompt_builders.py` 中精心编写的 4 个 SYSTEM_PROMPT (ROUTE/FEATURE_SEQUENCE/NODE_PARAMS/REPAIR) 和 4 个 `build_*_user_prompt` 函数 **全部未被调用**（死代码）。

`BasePackage.examples`（WASHER_EXAMPLE）和 `level2_usage_markdown` **从未注入 prompt**：
- `base_packages/axisymmetric/package.py` 第 240 行挂载 examples，但 grep 验证无人读取
- `skills/authoring_context.py` 的 `pack_authoring_context()` 函数专门为注入设计，但只在测试中使用
- `AuthoringContext.level2_usage_skills` 被构建、hash、存储，但在 `build_feature_sequence_tool_schema(ctx)` 中只用了 `ctx.selected_dialects`

**影响**：LLM 完全没有操作语义指导，只能靠字段名猜测。这导致 LLM 输出"区域描述"风格（bore_bottom/bore_wall/hub/web/rim）而非 handler 期望的"外轮廓连续段"。

### 根因层 2：Schema/Validation 不足

- `ProfileStation` 字段（`bases/axisymmetric/models.py` 第 12-24 行）**全部无 description**，LLM 只看到 `r_mm: any` 而非"半径（非直径）"
- `root_node` 可指向 `required=False` 节点（schema 无约束 + `_fix_root_node` 选最后一个 body 节点不看 required）
- `RimSlotProfile.validate_depth_order` 有约束但 autofix 无对应修复规则
- `_fix_profile_stations` 不检测"区域描述"风格（bore_bottom/bore_wall 等内壁站点 label）
- preflight 缺 bore-vs-profile 共面检测、revolve_profile 自相交检测

### 根因层 3：Handler 算法缺陷

- `handle_cut_annular_groove`（`handlers.py` 第 191-192 行）side 语义反转：`side="front"` 映射到 `bb.zmax`（应为 `zmin`）
- `handle_apply_safe_chamfer`（第 257-267 行）全局倒角策略对复杂几何必然失败，降级只有 distance/2，required=False 时仍报错而非 no-op
- `handle_revolve_profile`（第 31-87 行）sort-by-Z 算法对"区域描述"风格 **silently 产生错误形状**（不报错），自相交轮廓在 runtime 才暴露

### 根因层 4：配置错误

- `app/text-to-cad/vite.config.ts` 代理端口 8081，后端实际 8080

---

```python
def _fix_slot_station_order(doc: dict) -> dict:
    """修复 cut_rim_slot_pattern 的 slot_profile.stations depth 非递减顺序。"""
    for node in doc.get("nodes", []):
        if node.get("op") != "cut_rim_slot_pattern":
            continue
        sp = node.get("params", {}).get("slot_profile", {})
        if not isinstance(sp, dict):
            continue
        stations = sp.get("stations", [])
        if not stations or len(stations) < 2:
            continue
        depths = [s.get("depth_mm", 0) for s in stations]
        if depths != sorted(depths):
            sp["stations"] = sorted(stations, key=lambda s: s.get("depth_mm", 0))
    return doc
```

**4.2 增强 `_fix_root_node`**（第 522-544 行）：

修改选择逻辑，优先选 `required=True` 的 body 节点：
```python
def _fix_root_node(doc: dict) -> dict:
    """修正 root_node 引用。优先选 required=True 的最后 body 节点。"""
    node_ids = {n["id"] for n in doc.get("nodes", [])}
    node_by_comp: dict[str, list] = {}
    for n in doc.get("nodes", []):
        cid = n.get("component", "")
        node_by_comp.setdefault(cid, []).append(n)

    for comp in doc.get("components", []):
        rn = comp.get("root_node", "")
        if rn and rn in node_ids:
            # 检查当前 root_node 是否 required=False — 如果是则替换
            root_node_obj = next((n for n in node_by_comp.get(comp.get("id", ""), []) if n.get("id") == rn), None)
            if root_node_obj and root_node_obj.get("required", True):
                continue  # 当前 root_node 是 required=True，保留
            # 否则 fall through 重新选择
        cid = comp.get("id", "")
        comp_nodes = node_by_comp.get(cid, [])
        if comp_nodes:
            body_nodes = [n for n in comp_nodes
                          if any(o.get("name") == "body" and o.get("type") == "solid"
                                 for o in n.get("outputs", []))]
            # 优先 required=True 的 body 节点
            required_body_nodes = [n for n in body_nodes if n.get("required", True)]
            if required_body_nodes:
                comp["root_node"] = required_body_nodes[-1]["id"]
            elif body_nodes:
                comp["root_node"] = body_nodes[-1]["id"]
            else:
                comp["root_node"] = comp_nodes[-1]["id"]
    return doc
```

```python
    distance = float(node.typed_params.get("distance_mm", node.params.get("distance_mm", 0)))
    if distance > 0:
        target = node.params.get("target", "all_external_edges")
        try:
            body = _chamfer_by_target(body, distance, target)
        except Exception:
            try:
                body = _chamfer_by_target(body, distance / 2.0, target)
            except Exception:
                # 新增降级：required=False 时允许 no-op（保留原 body）
                if not node.required and node.degradation_policy == "may_skip_with_warning":
                    ctx.warnings.append(
                        f"Optional chamfer '{node.id}' skipped: "
                        f"failed at distance={distance}mm and {distance/2.0}mm"
                    )
                    ctx.degraded_features.append({
                        "node_id": node.id, "op": "apply_safe_chamfer",
                        "reason": f"chamfer failed at distance={distance}mm",
                    })
                    ctx.operation_metrics.append({
                        "node_id": node.id, "op": "apply_safe_chamfer",
                        "status": "degraded",
                        "reason": f"chamfer failed at distance={distance}mm",
                    })
                    # 返回原 body（不修改）
                    return {"body": _store_solid(node, ctx, body)}
                return handle_feature_failure(
                    node=node, ctx=ctx, original_body=body,
                    op_name="apply_safe_chamfer",
                    reason=f"chamfer failed at distance={distance}mm and fallback distance={distance/2.0}mm",
                )
    return {"body": _store_solid(node, ctx, body)}
```
