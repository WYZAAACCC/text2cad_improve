**文件：** `integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/runtime/geometry_postcheck.py`

**当前代码：**

**修复后：**
```python
# 5. Closed solid check
closed = _check_closed(solid)
is_valid = closed is True and volume is not None and volume > 0

# fail-closed: 无效实体必须阻断产出
# closed=None 表示无法检测（inspection artifact），不阻塞；closed=False 必须阻塞
if closed is False:

# is_valid_solid 综合校验：closed=True 且 volume>0
if closed is True and (volume is None or volume <= 0):
```

**说明：**

- `closed is False` 才阻塞（明确失败）；`closed is None`（无法检测）不阻塞
- `is_valid` 综合判断：closed=True 且 volume>0；若 closed=True 但 volume<=0 则矛盾，阻塞

**正向性：** 对所有零件都正向，fail-closed 是系统承诺。

```python
def _apply_fix(
    name: str, fix_fn,
    severity: Literal["safe_alias", "semantic_guess", "destructive"] = "safe_alias",
    confidence: float = 1.0,
):
    """Apply one fix and record all changes by comparing before/after hashes."""
    nonlocal doc
    # v6.2: 强制 category 检查 — 拒绝不在允许集合中的 fix
    cat = SEVERITY_TO_CATEGORY.get(severity, AutoFixCategory.SEMANTIC_GUESS)
    if cat not in DEFAULT_ALLOWED_AUTOFIX_CATEGORIES:
        entries.append(AutoFixEntry(
            rule_id=name,
            path="/",
            old_value="<skipped>",
            new_value="<skipped>",
            severity=severity,
            confidence=confidence,
            message=f"Fix '{name}' skipped: category '{severity}' not in allowed categories {sorted(c.value for c in DEFAULT_ALLOWED_AUTOFIX_CATEGORIES)}",
        ))
        return  # 不执行 fix
    before = stable_hash(doc)
    doc = fix_fn(doc)
    after = stable_hash(doc)
    if before != after:
        entries.append(AutoFixEntry(
            rule_id=name,
            path="/",
            old_value=f"<hash:{before[:12]}>",
            new_value=f"<hash:{after[:12]}>",
            severity=severity,
            confidence=confidence,
            message=f"Applied fix: {name}",
        ))
```

#### B1. 新增 A007：多 annular_groove z 区间冲突检测

**文件：** `integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/dialects/axisymmetric/dialect.py`
**位置：** `preflight_component` 函数，A004 之后（约 lines 247 之后，A005 之前）

**修复方案：**
在 A004 检查块结束后，A005 检查块开始前，插入 A007 检查：
```python
        # ── A007: 多 annular_groove z 区间冲突检测 ──
        # 检测多个 groove 的 z 区间是否重叠，避免几何冲突
        groove_nodes = [n for n in nodes if n.op == "cut_annular_groove"]
        groove_intervals: list[tuple[str, float, float]] = []  # (node_id, z_start, z_end)

        for g in groove_nodes:
            p = g.typed_params if hasattr(g, 'typed_params') and g.typed_params else g.params
            depth = p.get("depth_mm", 0)
            side = p.get("side", "front")
            z_position = p.get("z_position_mm")

            if z_position is None:
                # A007a: 缺少 z_position_mm 时警告（不阻塞，但提示风险）
                issues.append(ValidationIssue(
                    stage=stage,
                    code="a007_groove_z_position_missing",
                    message=(
                        f"cut_annular_groove '{g.id}' 缺少 z_position_mm，将默认使用端面位置。"
                        f"多个 groove 同时缺失 z_position_mm 会导致 z 区间重叠。"
                    ),
                    severity="warning",
                    node_id=g.id,
                ))
            else:
                # 有 z_position_mm：计算 z 区间
                try:
                    z_pos = float(z_position)
                    d = float(depth) if depth else 0.0
                    # side="front" → +Z 方向切割，z 区间 [z_pos, z_pos + d]
                    # side="rear"  → -Z 方向切割，z 区间 [z_pos - d, z_pos]
                    if side == "front":
                        z_start, z_end = z_pos, z_pos + d
                    else:
                        z_start, z_end = z_pos - d, z_pos
                    groove_intervals.append((g.id, z_start, z_end))
                except (TypeError, ValueError):
                    pass  # 参数类型错误由其他规则处理

        # A007b: 检测区间重叠
        for i, (id1, s1, e1) in enumerate(groove_intervals):
            for id2, s2, e2 in groove_intervals[i + 1:]:
                if s1 < e2 and s2 < e1:  # Z 区间严格重叠
                    issues.append(ValidationIssue(
                        stage=stage,
                        code="a007_groove_z_overlap",
                        message=(
                            f"cut_annular_groove '{id1}' (z={s1:.1f}→{e1:.1f}) 与 "
                            f"'{id2}' (z={s2:.1f}→{e2:.1f}) 的 z 区间重叠。"
                            f"请为每个 groove 指定不重叠的 z_position_mm。"
                        ),
                        severity="error",
                        node_id=id1,
                        expected={"non_overlapping_z_intervals": True},
                        actual={"groove_1": [round(s1, 2), round(e1, 2)], "groove_2": [round(s2, 2), round(e2, 2)]},
                    ))
```

**说明：**
- A007a 是 warning（缺少 z_position_mm 不阻塞，但提示风险）
- A007b 是 error（z 区间重叠必须阻塞）
- 只对有 z_position_mm 的 groove 做重叠检测

**正向性：** 对所有使用多 groove 的零件都正向（法兰多道密封槽、轴多道油槽等）。
**风险：** 低。新增规则，不修改现有逻辑。

```python
        # 检测伪中心细颈：r=1 或接近 0 的站点
        # 中心孔应由 cut_center_bore 处理，revolve_profile 只描述外轮廓
```

#### C4. 修复 TURBINE_DISC_REFERENCE_EXAMPLE 的 r=1 填充段

**文件：** `integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/base_packages/axisymmetric/package.py`
**位置：** lines 206-211（profile_stations 定义）

**当前代码（错误）：**
```python
"profile_stations": [
    {"r_mm": 1.0,  "z_front_mm": 0.0, "z_rear_mm": 5.0,  "label": "hub_lower_fill"},
    {"r_mm": 30.0, "z_front_mm": 5.0, "z_rear_mm": 35.0, "label": "hub_body"},
```

**问题分析：**
- r=1 的站点违反 A009（r < 5mm 警告）
- r=1 的站点是"伪中心细颈"，不符合外轮廓语义
- z 区间重叠（hub_lower_fill Z=0→5 与 rim_outer Z=0→40 重叠），但这是已有的"区域描述"风格问题

**修复后（正确）：**
```python
"profile_stations": [
    {"r_mm": 30.0, "z_front_mm": 0.0,  "z_rear_mm": 5.0,  "label": "hub_front"},
    {"r_mm": 30.0, "z_front_mm": 5.0,  "z_rear_mm": 35.0, "label": "hub_body"},
    {"r_mm": 100.0,"z_front_mm": 5.0,  "z_rear_mm": 35.0, "label": "rim_outer"},
    {"r_mm": 30.0, "z_front_mm": 35.0, "z_rear_mm": 40.0, "label": "hub_rear"},
],
```

**说明：**
- 去除 r=1 填充段，最小 r=30（轮毂外圆半径）
- 中心孔由 cut_center_bore 处理（diameter=60，r=30），与轮毂外圆一致
- profile_stations 描述外轮廓：轮毂（r=30）→ 轮缘（r=100）→ 轮毂（r=30）
- z 区间不重叠（A006 兼容）：
  - hub_front: Z=0→5
  - hub_body: Z=5→35
  - rim_outer: Z=5→35（与 hub_body 同 Z 范围，但 r 不同，这是台阶过渡）
  - hub_rear: Z=35→40

**注意：** rim_outer 与 hub_body 的 z 区间相同（Z=5→35），这会触发 A006 的 z 重叠检测。

**进一步修复（避免 A006 误报）：**
需要重新设计 profile_stations，确保 z 区间不重叠。涡轮盘的正确外轮廓应该是：
- Z=0→5: 轮毂前段（r=30）
- Z=5→35: 轮缘段（r=100）— 这是主体
- Z=35→40: 轮毂后段（r=30）

但这丢失了轮毂-轮缘的台阶过渡。实际上，A006 检测的是"区域描述"风格，而台阶过渡是合法的。

**最终修复方案（使用 4 段不重叠 z 区间）：**
```python
"profile_stations": [
    {"r_mm": 30.0, "z_front_mm": 0.0,  "z_rear_mm": 5.0,  "label": "hub_front"},
    {"r_mm": 100.0,"z_front_mm": 5.0,  "z_rear_mm": 35.0, "label": "rim_outer"},
    {"r_mm": 30.0, "z_front_mm": 35.0, "z_rear_mm": 40.0, "label": "hub_rear"},
],
```

**说明：**
- 3 段不重叠 z 区间：Z=0→5（轮毂前）, Z=5→35（轮缘）, Z=35→40（轮毂后）
- 最小 r=30（轮毂外圆），最大 r=100（轮缘外圆）
- 中心孔由 cut_center_bore 处理（diameter=60，r=30）
- A006 兼容（z 区间不重叠）
- A009 兼容（最小 r=30 >= 5mm）
- 体现涡轮盘的"轮毂-轮缘-轮毂"三段结构

**正向性：** 修复错误示例，对所有学习此示例的 LLM 都正向。
