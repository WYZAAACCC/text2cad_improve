# 涡轮盘分层建模架构修复计划

## Summary

test_v3 测试发现 LLM 试图在 `revolve_profile` 中用零宽度站点表达 Rim(250)→Web(210) 的瞬时跳变，被 A001 拦截。深度分析发现根本原因是 **TURBINE_DISC_REFERENCE_EXAMPLE 示范了错误的建模方式**（沙漏形外轮廓），且 `cut_annular_groove` 无法精确定位 Z 区间，导致 LLM 无法正确建模涡轮盘。

本计划通过 4 个修复项建立**正确的分层建模架构**：

| # | 修复项 | 位置 | 根因 |
|---|---|---|---|
| 1 | 扩展 cut_annular_groove 支持 z_position_mm | models.py + handlers.py | groove 语义限制 |
| 2 | 修正 TURBINE_DISC_REFERENCE_EXAMPLE 为分层建模 | package.py | example 错误示范 |
| 3 | 在 NODE_PARAMS_SYSTEM_PROMPT 添加 axisymmetric 规则 | prompt_builders.py | prompt 缺少建模规则 |
| 4 | 强化 ProfileStation validator + 新增 anti-example | models.py + package.py | validator 与 A001 不一致 |

**核心策略**：revolve_profile 只描述外轮廓（每个 Z 的最大半径），凹陷/侧面特征用 cut_annular_groove 独立操作创建。

## Current State Analysis

### 问题根因链

```
LLM 零宽度站点（A001 拦截）
    ↑
LLM 试图在 revolve_profile 中表达 Rim→Web 跳变
    ↑
LLM 学到了 TURBINE_DISC_REFERENCE_EXAMPLE 的错误模式
（示例展示"在 profile 中用半径变化表达 Web 内凹"）
    ↑
TURBINE_DISC_REFERENCE_EXAMPLE 本身是错误的建模方式
（沙漏形外轮廓，不是真正的涡轮盘）
    ↑
缺少正确的"分层建模"指导
（revolve 纯外轮廓 + cut_annular_groove 侧面切削）
    ↑
cut_annular_groove 语义限制
（无法精确指定 Z 区间，只能从端面切削 depth 毫米）
```

## Proposed Changes

### 修复 1：扩展 cut_annular_groove 支持 z_position_mm

#### 1.1 修改 CutAnnularGrooveParams

**文件**：`bases/axisymmetric/models.py`（[第 62-86 行](file:///e:/auto_detection_process/integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/bases/axisymmetric/models.py#L62-L86)）

**修改**：在 `depth_mm` 字段之后新增 `z_position_mm` 可选字段

```python
class CutAnnularGrooveParams(BaseModel):
    model_config = ConfigDict(extra="forbid")

    side: Literal["front", "rear"] = Field(
        description="groove 切削方向：front=向 +Z 方向切削，rear=向 -Z 方向切削。"
        "若 z_position_mm 缺失，front 从 zmin 端面开始，rear 从 zmax 端面开始。"
    )
    inner_dia_mm: DimExprOrFloat = Field(
        description="groove 内径（mm）。必须 < outer_dia_mm。"
    )
    outer_dia_mm: DimExprOrFloat = Field(
        description="groove 外径（mm）。必须 > inner_dia_mm。"
    )
    depth_mm: DimExprOrFloat = Field(
        description="groove 深度（mm）。从起始 Z 位置沿切削方向延伸的距离。"
    )
    z_position_mm: DimExprOrFloat | None = Field(
        default=None,
        description="可选，切削起始 Z 位置（mm）。"
        "指定后从该 Z 位置开始切削 depth_mm 毫米（side 决定方向）。"
        "缺失时使用 side 决定的端面（front=zmin, rear=zmax）。"
        "示例：z_position_mm=10, side=front, depth=20 → 切削 Z=10→30 的环形槽。"
    )

    @model_validator(mode="after")
    def validate_dia(self):
        # Skip comparison if either value is a DimExpr (dict) — the comparison
        # will be validated at analysis time after DimExpr resolution.
        if isinstance(self.inner_dia_mm, dict) or isinstance(self.outer_dia_mm, dict):
            return self
        if self.inner_dia_mm >= self.outer_dia_mm:
            raise ValueError("inner_dia_mm must be < outer_dia_mm")
        return self
```

**why**：增加可选的 z_position_mm 字段，支持精确定位 Z 区间。缺失时回退到当前行为，向后兼容。

#### 1.2 修改 handle_cut_annular_groove

**文件**：`dialects/axisymmetric/handlers.py`（[第 193-221 行](file:///e:/auto_detection_process/integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/dialects/axisymmetric/handlers.py#L193-L221)）

**修改**：在 `z_pos` 计算逻辑中支持 `z_position_mm`

```python
def handle_cut_annular_groove(node: CanonicalNode, ctx: RuntimeContext) -> dict[str, str]:
    import cadquery as cq
    body = resolve_input_object(node, ctx, 0)
    p = node.typed_params if node.typed_params else node.params
    inner = float(p.get("inner_dia_mm", 0))
    outer = float(p.get("outer_dia_mm", 0))
    depth = float(p.get("depth_mm", 0))
    side = p.get("side", "front")
    z_position = p.get("z_position_mm")
    if inner <= 0 or outer <= 0 or outer <= inner:
        return handle_feature_failure(
            node=node, ctx=ctx, original_body=body,
            op_name="cut_annular_groove",
            reason=f"invalid diameters: inner={inner}, outer={outer}",
        )
    try:
        bb = body.val().BoundingBox()
        # z_position_mm 优先；缺失时回退到 side 决定的端面
        if z_position is not None:
            z_pos = float(z_position)
        else:
            z_pos = bb.zmin if side == "front" else bb.zmax
        extrude_dir = depth if side == "front" else -depth
        ring = (
            cq.Workplane("XY").workplane(offset=z_pos)
            .circle(outer / 2.0).circle(inner / 2.0).extrude(extrude_dir)
        )
        result = body.cut(ring)
    except Exception as e:
        return handle_feature_failure(
            node=node, ctx=ctx, original_body=body,
            op_name="cut_annular_groove", exc=e,
        )
    return {"body": _store_solid(node, ctx, result)}
```

**why**：z_position_mm 优先；缺失时回退到 side 决定的端面（向后兼容）。side 仍然决定切削方向。

### 修复 3：在 NODE_PARAMS_SYSTEM_PROMPT 添加 axisymmetric 规则

**文件**：`authoring/prompt_builders.py`（[第 86-123 行](file:///e:/auto_detection_process/integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/authoring/prompt_builders.py#L86-L123)）

**修改**：在 `For shell:` 段落之后添加 axisymmetric 规则

```python
For axisymmetric revolve_profile:
- profile_stations describe the OUTER contour only — each Z position has exactly
  ONE radius (the maximum radius at that Z). Inner features (bore, web, hub) are
  created by separate cut operations, NOT by reducing r_mm in profile_stations.
- z_rear_mm must be STRICTLY GREATER than z_front_mm (zero-width stations are
  forbidden and will be rejected by validation).
- If a radius transition is needed (e.g., rim_to_hub), use a short but non-zero
  segment (>= 0.5mm) to represent the transition.
- Use cut_annular_groove with z_position_mm to machine side features at specific
  Z ranges (e.g., web recess from Z=10 to Z=30).
- DO NOT model an hourglass shape (narrow middle, wide ends) to represent web —
  this is incorrect for turbine discs. Use layered modeling instead.
```

**why**：明确告知 LLM axisymmetric 建模规则，引导正确的分层建模思路。

**文件**：`bases/axisymmetric/models.py`（[第 29-33 行](file:///e:/auto_detection_process/integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/bases/axisymmetric/models.py#L29-L33)）

```python
class ProfileStation(BaseModel):
    model_config = ConfigDict(extra="forbid")

    r_mm: DimExprOrFloat = Field(
        description="半径（RADIUS），不是直径。外径=2×r_mm。例如外径80mm的盘 r_mm=40."
    )
    z_front_mm: float = Field(
        description="该段起始 Z 位置（mm）。必须严格 < z_rear_mm（不允许零宽度段）。"
    )
    z_rear_mm: float = Field(
        description="该段结束 Z 位置（mm）。必须严格 > z_front_mm（不允许零宽度段）。"
    )
    label: str | None = Field(
        default=None,
        description="可选标签（如 'rim'/'hub'）。仅用于人类阅读，不影响几何。"
    )

    @model_validator(mode="after")
    def validate_z(self):
        if self.z_front_mm >= self.z_rear_mm:
            raise ValueError("z_front_mm must be strictly < z_rear_mm (zero-width stations forbidden)")
```

**why**：与 A001 保持一致，在 schema 层面就拦截零宽度站点，而不是等到 preflight 阶段。

## Assumptions & Decisions

### 假设
1. v0.2 dialect 系统是当前生产路径
2. `z_position_mm` 是可选字段，缺失时回退到当前行为（向后兼容）
3. ProfileStation validator 从 `>` 改为 `>=` 不会破坏现有合法输出（零宽度站点已被 A001 拦截）

### 决策
1. **cut_annular_groove 扩展方式**：增加可选 `z_position_mm` 参数（用户确认）。最小化修改，向后兼容。
2. **TURBINE_DISC_REFERENCE_EXAMPLE 修正方式**：改为分层建模（用户确认）。revolve 纯外轮廓 + cut_annular_groove 创建 Web 凹陷。
3. **ProfileStation validator**：从 `>` 改为 `>=`，与 A001 保持一致。在 schema 层面拦截零宽度站点。
4. **A001 不放宽**：保持 `zr > zf`（严格大于）。从 Prompt 层面引导 LLM 使用正确的建模方式。
5. **side 参数语义**：side 仍然决定切削方向（front=+z, rear=-z）。z_position_mm 只影响起始位置。
