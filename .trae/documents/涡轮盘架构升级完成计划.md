### 批次 5 剩余：Preflight Z 重叠检测

**目标**：在 runtime 之前检测"区域描述"风格，给出可操作的修复 hint。

**文件**：`integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/bases/axisymmetric/preflight.py`

**修改位置**：`preflight_revolve_profile` 函数（第 36-81 行），在现有检查之后、return 之前增加 Z 重叠检测。

**新增代码**：

```python
# 在 return issues 之前增加
# 检测"区域描述"风格：多个 station 的 Z 区间重叠
if len(stations) >= 2:
    z_ranges = [(s.get("z_front_mm", 0), s.get("z_rear_mm", 0)) for s in stations]
    overlap_count = 0
    for i, (zf1, zr1) in enumerate(z_ranges):
        for j, (zf2, zr2) in enumerate(z_ranges[i+1:], i+1):
            if zf1 < zr2 and zf2 < zr1:  # Z 区间重叠
                overlap_count += 1
    if overlap_count > 0:
        issues.append({
            "code": "revolve_profile_z_overlap",
            "message": (
                f"Profile stations have {overlap_count} overlapping Z ranges. "
                f"This indicates regional description style (bore/hub/web/rim), "
                f"which violates the Z single-valued outer contour constraint. "
                f"Convert to outer contour continuous segments where each Z position "
                f"has exactly one radius."
            ),
            "node_id": node.get("id"),
            "severity": "error",
        })
```

**验证**：用 `turbine_disc/llm_raw.json` 的 profile_stations 测试，确认 6 个 station 中有多对 Z 重叠时被检测到。

### 批次 6：Handler 算法修复

**目标**：修复 handler bug，增强降级策略，检测错误风格并 fail-closed。

**文件**：`integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/dialects/axisymmetric/handlers.py`

#### 6.1 修复 `handle_cut_annular_groove` side 语义反转

**修改位置**：第 191-192 行

```python
# 修改前（当前代码）
z_pos = bb.zmax if side == "front" else bb.zmin
extrude_dir = -depth if side == "front" else depth

# 修改后
z_pos = bb.zmin if side == "front" else bb.zmax
extrude_dir = depth if side == "front" else -depth
```

**语义说明**：
- `side="front"` = Z 较小端（zmin）：groove 从 zmin 向 +Z 方向切入，所以 `z_pos=zmin`, `extrude_dir=+depth`
- `side="rear"` = Z 较大端（zmax）：groove 从 zmax 向 -Z 方向切入，所以 `z_pos=zmax`, `extrude_dir=-depth`

**验证**：构造一个简单圆柱体，在 front 端切 groove，确认 groove 位于 zmin 侧。

#### 6.2 增强 `handle_revolve_profile` 检测"区域描述"风格

**修改位置**：第 55-62 行（多站点分支，`pts_2d.sort()` 之前）

```python
# 在 pts_2d 构建之后、sort 之前增加检测
# 检测"区域描述"风格：3+ stations 的 Z 区间完全重叠
if len(stations) >= 3:
    z_fronts = {s.get("z_front_mm", 0) for s in stations}
    z_rears = {s.get("z_rear_mm", 0) for s in stations}
    if len(z_fronts) == 1 and len(z_rears) == 1:
        raise ValueError(
            f"revolve_profile stations appear to be regional descriptions "
            f"(all share z_front={list(z_fronts)[0]}, z_rear={list(z_rears)[0]}). "
            f"Must be outer contour continuous segments (Z single-valued). "
            f"Cannot produce valid solid — requires LLM repair."
        )
```

**设计理由**：
- 这是第三道防线（prompt → preflight → handler）
- handler 是 fail-closed 的最后一关，必须 raise 而非 silent 产生错误形状
- ValueError 会被 `dialect.py` 的 `run_component` 捕获，若 node.required=False 则降级跳过，否则 raise 到 pipeline

**验证**：构造 3 个 station 共享相同 z_front/z_rear 的输入，确认 raise ValueError。

#### 6.3 改进 `handle_apply_safe_chamfer` 降级策略

**修改位置**：第 252-268 行

```python
# 修改后
def handle_apply_safe_chamfer(node: CanonicalNode, ctx: RuntimeContext) -> dict[str, str]:
    body = resolve_input_object(node, ctx, 0)
    distance = float(node.typed_params.get("distance_mm", node.params.get("distance_mm", 0)))
    if distance > 0:
        target = node.params.get("target", "all_external_edges")
        try:
            body = _chamfer_by_target(body, distance, target)
        except Exception:
            try:
                body = _chamfer_by_target(body, distance / 2.0, target)
            except Exception:
                # 降级：required=False 时允许 no-op（保留原 body）
                if not node.required and node.degradation_policy == "may_skip_with_warning":
                    ctx.warnings.append(
                        f"Optional chamfer '{node.id}' skipped: "
                        f"failed at distance={distance}mm and {distance/2.0}mm"
                    )
                    ctx.degraded_features.append({
                        "node_id": node.id, "op": "apply_safe_chamfer",
                        "reason": f"chamfer failed at distance={distance}mm",
                    })
                    ctx.operation_metrics.append({
                        "node_id": node.id, "op": "apply_safe_chamfer",
                        "status": "degraded",
                        "reason": f"chamfer failed at distance={distance}mm",
                    })
                    # 返回原 body（不修改）
                    return {"body": _store_solid(node, ctx, body)}
                return handle_feature_failure(
                    node=node, ctx=ctx, original_body=body,
                    op_name="apply_safe_chamfer",
                    reason=f"chamfer failed at distance={distance}mm and fallback distance={distance/2.0}mm",
                )
    return {"body": _store_solid(node, ctx, body)}
```

**设计理由**：
- 当前代码：chamfer 失败 → 调 `handle_feature_failure`，但该函数对 required=False 节点的处理依赖外层 `run_component` 的 try/except
- 问题：`handle_feature_failure` 内部可能 raise（取决于实现），导致 `run_component` 不得不捕获
- 改进：在 handler 内部直接判断 required + degradation_policy，若允许 skip 则直接返回原 body，避免异常传播
- 这让 chamfer 的降级策略更显式、更可控

**验证**：构造 required=False + degradation_policy="may_skip_with_warning" 的 chamfer 节点，触发 chamfer 失败，确认返回原 body 而非 raise。

## Assumptions & Decisions

### 已确认决策（用户此前选择，仍然有效）

1. **不改 IR 语义**：`ProfileStation` 仍只有 r_mm/z_front_mm/z_rear_mm/label 四个字段

- 不扩展 `handle_revolve_profile` 支持多区域截面（改 IR 语义，影响面大）
- 不重构 autofix 的 risk classification 系统
- 不修改 DeepSeek client 的 strict/tool_choice 配置
- 不修改 14 阶段 validation pipeline 结构
