你是一个资深前端开发工程师，专精于 React、TypeScript、Three.js 和复杂交互界面的实现。  
请根据下述需求生成一个完整的 **Text-to-CAD** 工具前端页面，要求代码可运行、结构清晰、注释充分。

## 一、项目概述
开发一个基于 Client?Server 架构的 Web 应用，用户通过自然语言描述生成 3D CAD 模型，并支持数据集管理（保存/浏览/复用历史设计）。

## 二、技术栈强制要求
- React 18 + TypeScript
- 3D 渲染：Three.js + @react-three/fiber + @react-three/drei
- 状态管理：Zustand
- 样式：TailwindCSS（暗色主题）
- HTTP：Axios
- 异步状态：React Query
- 图标：Lucide React
- 布局：Radix UI 的 Collapsible、Tabs 等（可用任意方式模拟）

## 三、整体布局与组件划分
页面采用三栏布局（可调宽度，但初始比例固定）：
- **左侧对话历史区**：展示用户消息、系统回复、模型卡片（带缩略图）。支持折叠。
- **中央 3D 视图区**：Three.js 场景，支持轨道控制（鼠标拖拽旋转、右键平移、滚轮缩放）、网格辅助、环境光/定向光。可显示线框/材质切换。
- **右侧面板**：使用 Tab 切换两个视图：`属性/图层` 和 `数据集管理`。面板支持折叠。

所有区域需实现自适应高度/宽度，暗色背景（#0f0f12 左右）。

## 四、详细功能要求

### 4.1 对话历史区
- 模拟聊天记录：显示用户消息（右对齐）和系统回复（左对齐）。
- 系统回复中应包含“模型卡片”：卡片上有缩略图（占位图即可，后期可替换为真实数据）、模型名称，以及“加载”、“删除”按钮。
- 每条消息下方有操作菜单（复制文本、重新生成）。
- 支持清空对话（顶部“新建对话”按钮）。
- 对话数据存储在 Zustand 中。

### 4.2 3D 视图区
- 使用 `<Canvas>` 创建场景，默认显示一个基础立方体（用于演示，后续可通过 API 替换）。
- 提供顶部浮层工具栏：视图重置、切换线框/材质模式、导出截图。
- 当前模型中某个物体被选中时（点击模型），右侧属性面板显示其参数（位置、尺寸等）。
- 支持从数据集加载模型（glTF 或内置几何体）替换当前场景。

### 4.3 右侧面板 – 属性/图层
- 展示当前选中模型的参数（名称、类型、长宽高/半径、体积、表面积）。
- 提供滑块/输入框实时修改尺寸（简单的前端变换即可）。
- 显示场景中所有模型列表（图层），每个模型可控制可见性、删除。

### 4.4 右侧面板 – 数据集管理（重点）
- 功能：展示已保存的数据集条目（缩略图网格），支持添加当前设计、查看详情。
- 数据结构（前端模拟或使用 mock 数据，但需要模拟 API 调用）：
  - 每个条目包含：id、名称、缩略图URL、标签、以及 `data` 对象（包含 userInput, featureGraph, gcadDocument, cadQueryCode, stepFileUrl, solveResult）。
- **缩略图网格区**：响应式网格，每个卡片显示缩略图、名称、标签。单击卡片选中，双击卡片加载模型到 3D 视图。
- **“添加到数据集”按钮**：
  - 点击后弹窗，要求输入名称和标签。
  - 收集当前 3D 视图中的模型信息（简化为：当前模型的描述文本、模拟的 featureGraph 等）。
  - 将新条目加入 mock 数据集列表（前端模拟 POST 请求）。
- **详情分栏区**（选中某个条目后显示在下半部分）：
  - 使用子标签页显示六个字段：用户输入、FeatureGraph、GCAD文档、CADQuery代码、STEP文件、求解结果。
  - 每个字段的内容展示：
    - 用户输入：纯文本，可复制。
    - FeatureGraph：简化为 JSON 树形视图或简单的节点-边说明。
    - GCAD文档：代码框（带语法高亮或纯文本），可下载。
    - CADQuery代码：同左，可复制。
    - STEP文件：显示文件大小并提供下载按钮（模拟链接）。
    - 求解结果：表格展示体积、表面积、质心等。
  - 提供“加载到场景”按钮，将该条目的模型（模拟 glTF 路径）加载到 3D 视图。

### 4.5 后端交互模拟
由于是前端原型，所有后端 API 用 mock 实现（setTimeout 延迟）。但需保留真实的 Axios 请求结构，以便后续接入真实后端。
需 mock 下列接口：
- `POST /api/generate` – 接收 {text, sessionId}，返回 taskId，随后轮询状态返回 mock 模型数据（glTF 文件路径）。
- `GET /api/dataset/list` – 返回 mock 条目列表。
- `POST /api/dataset/entry` – 添加新条目。
- `GET /api/dataset/entry/{id}` – 获取详情数据。

### 4.6 交互流程示例
- 用户在底部输入框输入“生成一个半径为2的球体”，点击发送 → 显示加载状态 → 轮询完成后，3D 视图显示球体，对话区增加系统回复（带模型卡片）。
- 用户点击右侧“数据集管理” → 点击“添加到数据集” → 输入名称 → 新卡片出现在网格中。
- 用户单击刚保存的卡片 → 下方详情区显示所有字段内容。
- 用户点击“加载到场景” → 3D 视图切换为该模型。
