我先给结论：**这次 `turbine_disc/test_v11` 的问题不是单纯“LLM 画得不好”，而是“示例/路由/基础 primitive 能力边界/几何后验校验”同时有问题。** 当前系统把一个高压涡轮盘错误地简化成了“旋转圆盘 + 圆孔 + 外缘齿槽切削”，但真实涡轮盘至少需要表达 **hub/web/rim 的轴向厚度随半径变化** 和 **外缘轴向贯通的枞树形榫槽/轮槽**。test_v11 目录本身包含 `route_plan.json`、`llm_raw.json`、`raw_fixed.json`、`output.metadata.json`、`validation_report.json` 和渲染图等输出文件，说明这不是一次临时失败，而是一个完整的测试链路已经把错误几何当成“通过”产物保存下来了。([GitHub][1])

## 1. 正确涡轮盘结构应该怎么理解

航空发动机涡轮盘不是普通平板齿轮盘。典型结构应按 **中心孔/轮毂 hub → 过渡腹板 web → 外缘 rim → 榫槽/轮槽 fir-tree slots** 来理解。公开资料中也把燃气涡轮转子盘描述为：中心附近为环形 hub，web 径向向外延伸，外端为 annular rim，rim 上有多个轴向贯通的 dovetail/fir-tree slots，用来安装叶片根部。([谷歌专利][2])

因此你说的“靠近中间位置厚度大，厚度沿着半径方向递减，最后在榫槽处增大以安置榫槽”是正确方向。它的剖面不是 test_v11 现在那种上下一刀切的阶梯状塔形，也不是普通齿轮盘；更像是一个**中心轮毂较厚、腹板向外减薄、外缘局部加厚的轴对称承力盘**。NASA 的转盘截面示意也把 disk cross section 分成 bore、web、rim 等多个径向 section，并以不同半径和厚度参数表达。([国家航空航天局技术报告服务器][3])

榫槽方面，真实枞树槽不是单纯锯齿口。fir-tree 是多瓣/多齿形连接结构，用于把叶片根部嵌入盘缘；资料明确称 fir-tree 是用于将叶片装入盘的 multilobed feature，叶片根与盘槽通过互补轮廓啮合。([科学直通车][4]) 另有资料说明，盘缘上的 fir-tree mounting slots 通常通过拉削、线切割或铣削形成，盘槽边缘还需要去毛刺和定义圆角，这说明真实槽型应包含齿肩、颈部、接触斜面和圆角，而不是尖锐折线。([otec.de][5])

## 2. test_v11 实际哪里错了

`route_plan.json` 已经暴露了第一层问题：系统识别到对象是 `axisymmetric_turbine_disk`，并选择了 `deterministic_primitive` 和 `axisymmetric_turbine_disk`，但同一份 route plan 又明确列出当前 primitive 不支持“多级 fir-tree，stage 宽度交替变化”的表达，也不支持两个不同直径范围的前侧 seal grooves。也就是说，路由器已经知道能力不足，却仍然放行到确定性 primitive，而不是 fail-closed 或切换到更通用的 profile/sweep 构建路径。([GitHub][6])

`llm_raw.json` 的第二层错误更关键：它把基础盘体写成 `revolve_profile`，但 `profile_stations` 是 `{r_mm, z_front_mm, z_rear_mm}` 这种“半径随 Z 分段”的结构，例如从 `r=1` 到 `r=130`、`r=210`、`r=250` 再回到 `r=130`、`r=1`。这实际上是在沿轴向 Z 排布多个半径段，结果更像“轴向拉长的异形柱体/塔形剖面”，而不是“半径方向上厚度变化的涡轮盘”。([GitHub][7])

源码也能解释为什么会错。`RevolveProfileParams` 的说明要求 profile 必须是 **Z 的单值函数**，同一 Z 只能有一个半径，而且提示不能按 bore/hub/web/rim 区域输出；但真实涡轮盘剖面需要表达的是 **轴向厚度 t 随半径 r 变化**，即前后端面是 `z_front(r)` / `z_rear(r)`，这天然不是简单的 `r(z)` 单值外轮廓。当前 schema 可以做简单旋转体和台阶圆柱，但不适合高保真涡轮盘剖面。([GitHub][8])

第三层错误在枞树槽。`cut_rim_slot_pattern` 的参数只有 `count`、`slot_depth_mm` 和 `slot_profile.stations[{depth_mm, half_width_mm}]`，源码实现是在 XY 平面用径向深度和半宽生成一个对称多边形 cutter，然后沿 Z 方向 extrude，旋转复制切外缘。这能做“外缘周期槽”，但不能表达真实枞树槽的齿面角、承力接触面、圆角半径、齿根圆弧、轮缘柱/榫齿工作面和非工作面等细节。([GitHub][9]) 你给的第一张图里有“榫齿工作面/非工作面/顶部”等分区，第二张图表现的是典型多级收放的枞树状根部/槽型轮廓；test_v11 的 `stations=[8,5,7,3]` 只是折线宽窄变化，不能代表正确枞树榫槽。([GitHub][7])

第四层错误是验证没有拦住错误结果。`validation_report.json` 显示 core validation 完全通过，运行了 structure、registry、params、graph、typecheck、phase、safety、canonicalize、dialect_semantics、geometry_preflight 等阶段，并给出 `ok: true`。([GitHub][10]) 但 `output.metadata.json` 又显示 `inspection_validation` 缺失，`geometry_postcheck` 中 `closed=false`、`is_valid_solid=false`，这与约束里的 `require_closed_solid=true` 明显冲突。也就是说，系统的 schema/graph 验证通过了，但真实几何质量没有被强制作为最终失败条件。([GitHub][11])

所以责任划分应该是：

```text
不是单纯 LLM 错。
而是：
1. 示例 prompt/测试目标没有把真实涡轮盘剖面和枞树槽定义清楚；
2. route_plan 明知 primitive 表达力不足仍选择 deterministic_primitive；
3. axisymmetric revolve_profile 的 schema 不适合真实 turbine disk t(r) 剖面；
4. cut_rim_slot_pattern 只能做简化折线槽，不能做工程意义上的 fir-tree slot；
5. validation 只验证 IR 合法性，没有把“几何形状语义正确性”和“solid 有效性”作为硬门槛。
```

## 3. 重新写一份正确的涡轮盘建模 prompt

下面这份 prompt 的目标不是制造级设计，而是让 text-to-cad 系统生成**形状正确的参考几何**。关键是强制它不要把涡轮盘建成普通阶梯圆盘，也不要把榫槽建成普通锯齿槽。

```text
请生成一个“航空发动机高压涡轮盘 reference geometry”，单位为 mm，仅用于非飞行、非认证、非制造的概念参考模型。不要声明 airworthy、certified、manufacturing-ready、structurally validated 或 fatigue/life prediction。

几何目标：
1. 整体为单一闭合实体 turbine disk，绕 Z 轴旋转成型，Z 轴为发动机轴向，R 方向为半径方向。
2. 盘体必须包含中心孔 bore、厚轮毂 hub、减薄腹板 web、加厚外缘 rim 和外缘轴向贯通 fir-tree blade-root slots。
3. 盘体剖面必须按“轴向厚度随半径变化”建模，而不是按 Z 方向堆叠圆柱段：
   - 中心孔直径约 120 mm。
   - hub 区域半径约 60–120 mm，轴向厚度最大，约 70–80 mm。
   - web 区域半径约 120–215 mm，轴向厚度沿半径方向平滑或分段递减，外侧厚度约 28–35 mm。
   - rim 区域半径约 215–250 mm，为安置榫槽重新加厚，轴向厚度约 55–65 mm。
   - hub→web、web→rim 之间必须有圆角/斜面过渡，不允许形成单薄尖角或不合理竖直塔形。
4. 基础旋转剖面应表现为：中心轮毂厚、向外腹板减薄、外缘加厚；从侧视图看应类似真实涡轮盘截面，而不是平板、塔形件或普通齿轮。
5. 外缘必须生成 60 个左右均布的轴向贯通 fir-tree slots，槽沿 Z 方向贯穿 rim，槽开口位于外圆周，槽型为雌槽。
6. fir-tree slot 截面必须是多级枞树形，而不是普通矩形槽、三角槽、锯齿槽或简单 dovetail：
   - 从外圆向内依次包含入口颈部、第一对齿肩/承力瓣、第二颈部、第二对齿肩/承力瓣、第三颈部、底部圆角槽根。
   - 宽度必须呈“窄颈—宽齿肩—窄颈—宽齿肩—窄底部/圆角根部”的交替变化。
   - 左右两侧应镜像对称，齿面为倾斜承力面，所有齿根和齿顶转角必须用小圆角或倒圆近似，不能保留尖锐内角。
   - slot depth 约 18–24 mm；slot mouth 半宽约 3–4 mm；第一齿肩半宽约 7–8 mm；第二齿肩半宽约 6–7 mm；底部半宽约 3–4 mm。
7. rim 上每两个槽之间保留轮缘柱/盘齿 posts，不能把外缘切穿成断裂碎片。
8. 可加入前后端面轻微环形台阶或密封槽，但不得让这些槽破坏 hub/web/rim 主剖面。
9. 输出必须是一个 closed solid，body_count=1，STEP + metadata sidecar 均必须生成。
10. 如果当前 primitive 不能表达“半径方向厚度变化剖面”或“多级圆角 fir-tree slot”，不要用过度简化几何冒充正确涡轮盘；应 fail-closed，并在 unsupported_capabilities 中明确说明需要：
    - arbitrary ordered R-Z revolve polygon / thickness-by-radius disk profile；
    - fir_tree_slot_v2 with alternating neck/lobe stations, flank angles, and fillet radii；
    - final geometry_postcheck enforcing closed=true and is_valid_solid=true。
```

如果你要把它进一步压缩成系统更容易吃的英文版，可以用这一版：

```text
Create a non-flight, non-certified reference geometry of an aero-engine high-pressure turbine disk in millimeters.

The part must be a single closed solid with a central bore, thick hub, tapered web, thickened rim, and axial through fir-tree blade-root slots around the rim. The Z axis is the engine axis; radial direction is R.

Do not model the disk as a simple flat plate, gear, or stack of axial cylinders. The side cross-section must represent axial thickness as a function of radius: thick near the hub, gradually thinner through the web, and thicker again at the rim to carry the blade-root slots. Use smooth blends or chamfers between hub, web, and rim.

Approximate dimensions:
- outer diameter about 500 mm;
- center bore diameter about 120 mm;
- hub radius 60–120 mm, maximum axial thickness about 70–80 mm;
- web radius 120–215 mm, thickness tapering down to about 28–35 mm;
- rim radius 215–250 mm, thickened to about 55–65 mm.

Add about 60 equally spaced axial through fir-tree slots at the outer rim. Each slot must be a female fir-tree groove, not a rectangular, triangular, saw-tooth, or simple dovetail slot. The slot profile must alternate narrow necks and wider lobes: mouth neck, first load lobe, second neck, second load lobe, lower neck/root with rounded bottom. Use symmetric inclined flanks and small fillets at tooth roots and lobe transitions.

If the current CAD primitive cannot represent thickness-by-radius disk profiles or multi-lobed filleted fir-tree slots, fail closed instead of producing an oversimplified approximation. Report the missing capabilities explicitly: arbitrary ordered R-Z revolve polygon and fir_tree_slot_v2 with neck/lobe widths, flank angles, and fillet radii.
```

## 4. 系统该怎么修

第一，`route_plan` 不能一边写 `unsupported_capabilities`，一边继续选择 `deterministic_primitive`。对于这类涡轮盘，应增加硬规则：只要用户要求“真实涡轮盘剖面 + 枞树槽”，而当前 primitive 不能表达多级榫槽或厚度随半径变化，就必须 `route_decision = fail_closed` 或切换到 `custom_ordered_revolve_profile + fir_tree_slot_v2`，不能降级成普通外缘槽。test_v11 的 route plan 已经识别出 fir-tree 和 seal groove 表达能力不足，但仍放行，这是路由层错误。([GitHub][6])

第二，`revolve_profile` 需要升级。当前 `ProfileStation` 是 `r_mm + z_front_mm + z_rear_mm`，并且要求 profile 是 Z 单值函数；这非常适合简单轴类/盘类台阶外轮廓，但不适合真实 turbine disk 的 R-Z 闭合截面。应新增一种 profile 类型，例如：

```json
{
  "op": "revolve_closed_rz_polygon",
  "params": {
    "axis": "Z",
    "points_rz_mm": [
      [60, -38], [120, -38], [170, -22], [215, -16], [250, -32],
      [250,  32], [215,  16], [170,  22], [120,  38], [60,  38]
    ],
    "inner_bore_radius_mm": 60,
    "blend_policy": "safe_fillets_optional"
  }
}
```

也就是说，必须保留 R-Z 多边形点顺序，而不是按 Z 排序。源码中当前处理逻辑会把点按 Z 排序，这会破坏真实涡轮盘剖面的拓扑顺序。([GitHub][9])

第三，`cut_rim_slot_pattern` 需要升级为 `cut_fir_tree_slot_pattern_v2`。当前参数只有 `depth_mm` 和 `half_width_mm`，没有齿面角、圆角半径、齿肩/颈部语义，也无法表达工作面/非工作面。建议 schema 改成：

```json
{
  "op": "cut_fir_tree_slot_pattern_v2",
  "params": {
    "count": 60,
    "slot_axis": "Z",
    "rim_outer_radius_mm": 250,
    "slot_total_depth_mm": 22,
    "stages": [
      {"name": "mouth_neck", "depth_mm": 0,  "half_width_mm": 3.5, "fillet_mm": 0.8},
      {"name": "upper_lobe",  "depth_mm": 5,  "half_width_mm": 8.0, "flank_angle_deg": 35, "fillet_mm": 1.2},
      {"name": "mid_neck",    "depth_mm": 9,  "half_width_mm": 4.5, "fillet_mm": 1.0},
      {"name": "lower_lobe",  "depth_mm": 14, "half_width_mm": 7.0, "flank_angle_deg": 35, "fillet_mm": 1.2},
      {"name": "root_neck",   "depth_mm": 19, "half_width_mm": 4.0, "fillet_mm": 1.0},
      {"name": "rounded_root","depth_mm": 22, "half_width_mm": 3.5, "root_radius_mm": 2.0}
    ],
    "require_smooth_internal_corners": true
  }
}
```

第四，最终几何检查必须收紧。`output.metadata.json` 里已经出现 `closed=false` 和 `is_valid_solid=false`，但构建仍产出 output，这在零信任 CAD 架构下不应该被接受。只要用户约束里有 `require_closed_solid=true`，最终 metadata 中 `geometry_postcheck.closed=false` 或 `is_valid_solid=false` 就必须让构建失败，而不是只写入 metadata。([GitHub][11])

## 5. 最终判断

这次失败的根因排序如下：

```text
第一责任：示例/测试 prompt 不够工程化，没有明确真实涡轮盘剖面和枞树槽语义。
第二责任：路由器错误，明知 primitive 不支持关键能力仍然选择 deterministic_primitive。
第三责任：axisymmetric primitive 表达力不足，不能高保真表达 turbine disk 的 t(r) 剖面。
第四责任：fir-tree slot primitive 过于简化，只能切折线外缘槽。
第五责任：验证系统只证明“IR 合法”，没有证明“几何语义正确”和“最终实体有效”。
```

因此，**示例本身有问题，LLM 的建模选择也有问题，但更根本的是当前系统缺少适合涡轮盘的专用几何语义 primitive。** 仅靠修改 prompt 可以显著改善，但如果不新增 `ordered R-Z closed revolve profile` 和 `fir_tree_slot_v2`，系统仍然很容易生成“看起来有槽但不是航空涡轮盘”的伪正确模型。
