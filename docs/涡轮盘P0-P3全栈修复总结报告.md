## 一、问题背景

### 1.1 故障全景

涡轮盘建模测试暴露了系统在多个层面的缺陷，从 Prompt 到验证再到几何内核，形成了完整的故障链：

| 层面 | 问题 | 表现 |
|------|------|------|
| Prompt | LLM 输出"区域描述"风格 profile | bore/hub/web/rim 分段，Z 重叠，语义断层 |
| Prompt | LLM 误解 r=1 填充段用途 | 产生 degenerate needle solid（A009 拦截） |
| Validation | geometry_postcheck 未纳入 import_gate | 开放实体可能通过 gate（fail-open 风险） |
| Validation | preflight 缺多 groove Z 区间冲突检测 | 多个 cut_annular_groove 互相覆盖 |
| Validation | preflight 缺最小半径校验 | r<0.5mm 的针状实体漏检 |
| Handler | cut_annular_groove side 语义反转 | front/rear 与 Z 方向映射错误 |
| Handler | chamfer 全局策略对复杂几何必然失败 | 60 个榫槽后 600+ 条边倒角崩溃 |

### 1.2 核心设计思想：三层防御

本次修复遵循"Prompt → Validation → Handler"三层防御架构：

```
┌─────────────────────────────────────────────────────────┐
│  Layer 1: Prompt 层（C1-C4）                            │
│  - IR 能力边界说明                                       │
│  - trust_level 与安全声明                                │
│  - anti_examples 反向注入                                │
│  - reference example 正确示范                            │
└──────────────┬──────────────────────────────────────────┘
               ▼
┌─────────────────────────────────────────────────────────┐
│  Layer 2: Validation 层（P0 + P1 = A1-A4 + B1-B2）      │
│  - geometry_postcheck ok 判定含 closed/is_valid         │
│  - auto_fixer category 强制检查                         │
│  - unsupported_capabilities 流转                        │
│  - geometry_postcheck 纳入 import_gate                  │
│  - A007 多 groove Z 区间冲突检测                         │
│  - A009 最小半径校验                                     │
└──────────────┬──────────────────────────────────────────┘
               ▼
┌─────────────────────────────────────────────────────────┐
│  Layer 3: Handler 层（已在之前轮次修复）                  │
│  - cut_annular_groove side 语义修正                      │
│  - cut_rim_slot_pattern count 阈值策略切换               │
│  - chamfer 降级策略优化                                  │
└─────────────────────────────────────────────────────────┘
```

每一层都能独立拦截错误，上层拦截成本最低，下层拦截是最后防线。

## 三、P0：架构 Fail-Closed 加固（A1-A4）

### A1：geometry_postcheck ok 判定含 closed/is_valid

**问题**：`geometry_postcheck` 的 `ok` 判定之前可能仅检查了运行时异常，未确保最终实体是闭合且有效的。若产生开放壳体或零体积实体，`ok` 仍可能为 True，存在 fail-open 风险。

**修复**：在 `geometry_postcheck.py` 中强化 `ok` 判定逻辑：

- `closed` 检查：最终实体必须是闭合的（B-Rep 拓扑完整）
- `is_valid` 综合校验：`closed=True` 且 `volume > 0`
- `closed=None`（inspection artifact 无法检测）不阻塞
- `closed=False` 必须阻塞，添加错误：`"Final solid is not closed (B-Rep topology broken)"`
- `closed=True` 但 `volume<=0` 也阻塞，添加错误：`"Final solid is invalid (closed=True but volume is non-positive)"`
- `ok = len(errors) == 0`

**文件**：[geometry_postcheck.py](file:///e:/auto_detection_process/integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/runtime/geometry_postcheck.py#L125-L137)

**意义**：确保 import_gate 的几何后置检查是真正可靠的，不会让无效实体通过。

---

### A2：auto_fixer SEVERITY_TO_CATEGORY + category 强制检查

**问题**：auto_fixer 的 severity 字符串（如 `"safe_alias"`, `"semantic_guess"`, `"destructive"`）与 `AutoFixCategory` 枚举之间缺少强制映射和检查。如果某个 fix 的 severity 不在允许的 categories 中，可能仍被执行，导致过度修复或不安全修复。

**修复**：
1. 新增 `SEVERITY_TO_CATEGORY` 映射表：
   - `"safe_alias"` → `AutoFixCategory.CONTEXT_SAFE`
   - `"semantic_guess"` → `AutoFixCategory.SEMANTIC_GUESS`
   - `"destructive"` → `AutoFixCategory.DESTRUCTIVE`
2. 在 `_apply_fix` 中增加强制 category 检查：
   - 映射失败时默认到 `SEMANTIC_GUESS`（保守降级）
   - 若 category 不在 `DEFAULT_ALLOWED_AUTOFIX_CATEGORIES` 中，跳过该 fix 并记录警告
   - 不静默执行，确保 fail-closed

**文件**：[auto_fixer.py](file:///e:/auto_detection_process/integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/authoring/auto_fixer.py#L87-L92) + [auto_fixer.py](file:///e:/auto_detection_process/integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/authoring/auto_fixer.py#L235-L267)

**意义**：auto_fixer 不会擅自执行不允许的修复类别，保证修复安全性可控。

### A3：unsupported_capabilities 全链路流转

**问题**：当 LLM 生成的操作包含系统不支持的能力时，缺少一个标准化的字段来记录和传递这些信息。不支持的能力可能被静默忽略，也可能在后续环节导致难以诊断的错误。

**修复**：
1. `metadata_v3.py` 中 `GenerativeMetadataV3` 模型新增 `unsupported_capabilities: list[str]` 字段
2. `build_generative_metadata_v3` 函数增加 `unsupported_capabilities` 参数
3. `run.py` 中调用 `build_generative_metadata_v3` 时传入 `unsupported_capabilities`，从 canonical document 中提取
4. 空值安全处理：`list(unsupported_capabilities) if unsupported_capabilities else []`

**文件**：[metadata_v3.py](file:///e:/auto_detection_process/integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/pipeline/metadata_v3.py#L82-L165) + [run.py](file:///e:/auto_detection_process/integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/pipeline/run.py#L361-L369)

**意义**：不支持的能力被显式记录和传递，便于诊断和下游处理，避免静默失败。

### A4：geometry_postcheck_ok 纳入 import_gate

**问题**：import_gate 的 `REQUIRED_GATE_FLAGS` 之前不含 `geometry_postcheck_ok`。这意味着即使几何后置检查失败，产出物仍可能被标记为可导入，存在严重的 fail-open 安全漏洞。

**修复**：4 处修改确保 geometry_postcheck 是 import gate 的硬条件：

1. `REQUIRED_GATE_FLAGS` 列表增加 `"geometry_postcheck_ok"`（共 13 个 flag）
2. `gate` dict 初始化增加 `"geometry_postcheck_ok": False`（fail-closed 默认值）
3. 实际检查逻辑：从 validation dict 中读取 `geometry_postcheck.ok`，若非 True 则立即返回失败，issue code 为 `"geometry_postcheck_not_ok"`
4. `REQUIRED_TRUE_FLAGS` 增加 `"geometry_postcheck_ok"`

**文件**：[import_artifact.py](file:///e:/auto_detection_process/integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/pipeline/import_artifact.py#L12-L190)

**关联（A4-1）**：`REQUIRED_VALIDATION_STAGES` 增加 `"geometry_postcheck"`，确保 metadata 验证时检查该 stage 是否存在。
**文件**：[metadata.py](file:///e:/auto_detection_process/integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/pipeline/metadata.py#L9-L16)

**意义**：这是 P0 最关键的架构修复——几何后置检查从"可选项"变为"硬门禁"，彻底关闭 fail-open 漏洞。

## 四、P1：Preflight 校验增强（B1-B2）

### B1：A007 多 groove Z 区间冲突检测

**问题**：当模型包含多个 `cut_annular_groove` 操作时，preflight 不检查它们的 Z 区间是否重叠。如果 LLM 生成的多个 groove 在 Z 方向上互相覆盖，会导致几何结果不可预期（重复切割、布尔操作失败等）。

**修复**：在 `dialect.py` 的 `preflight_component` 中新增 A007 检测：

1. 收集所有 `cut_annular_groove` 节点
2. 对每个 groove，根据 `side` 参数计算其 Z 区间：
   - `side="front"` → zmin 方向切割，区间为 `[z_position - depth, z_position]`
   - `side="rear"` → zmax 方向切割，区间为 `[z_position, z_position + depth]`
   - 未指定 `z_position_mm` 时回退到 side + depth 的默认位置
3. 检测任意两个 groove 之间的严格重叠（`zf1 < zr2 and zf2 < zr1`）
4. 发现重叠时生成 `a007_groove_z_overlap` 错误，包含重叠的两个 groove 节点 ID 和重叠区间

**文件**：[dialect.py](file:///e:/auto_detection_process/integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/dialects/axisymmetric/dialect.py#L277-L314)

**意义**：作为第二层防御，在几何构建前就拦截明显的 Z 区间冲突错误，减少 runtime 失败。

---

### B2：A009 revolve_profile 最小半径校验

**问题**：LLM 有时会生成极小半径（如 r=1mm）的 profile station 作为"填充段"或"连接段"。这些极小半径的站点会产生针状实体（degenerate needle solid），导致几何内核不稳定甚至崩溃。preflight 之前缺少这类检查。

**修复**：在 `dialect.py` 的 `preflight_component` 中新增 A009 检测：

1. 定义 `MIN_PROFILE_RADIUS = 0.5  # mm`
2. 遍历所有 `revolve_profile` 节点的 `profile_stations`
3. 对每个 station 的 `r_mm`：
   - 若是 dict 类型（DimExpr 符号表达式），跳过数值检查
   - 若是数值且 `0 < r < MIN_PROFILE_RADIUS`，生成 `a009_radius_too_small` 错误
   - 错误信息包含 station 索引、r 值和阈值
4. r=0（中心轴）是合法的，不拦截

**文件**：[dialect.py](file:///e:/auto_detection_process/integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/dialects/axisymmetric/dialect.py#L146-L172)

**意义**：从源头拦截会产生 degenerate 几何的极小半径，减少几何内核崩溃风险。

## 五、P3：Prompt 重设计与示例修正（C1-C4）

### C1：IR 能力边界说明

**问题**：LLM 在生成 G-CAD IR 时，不清楚系统的能力边界，可能尝试生成超出 axisymmetric 方言支持范围的操作（如非旋转对称特征、内部腔体等）。

**修复**：在 `NODE_PARAMS_SYSTEM_PROMPT` 末尾增加 **"IR CAPABILITY BOUNDARIES (must respect):"** 章节：

- axisymmetric 方言仅支持旋转对称体（绕 Z 轴旋转 360°）
- `revolve_profile` 仅描述外轮廓（outer contour），凹陷/侧面特征需用独立操作（cut_annular_groove 等）
- `cut_rim_slot_pattern` 仅在外圆周上开槽
- 列出支持的 7 个操作及其用途

**文件**：[prompt_builders.py](file:///e:/auto_detection_process/integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/authoring/prompt_builders.py#L145-L157)

**意义**：从 Prompt 层减少 LLM 的越界尝试，降低 validation 和 runtime 的拦截压力。

### C2：trust_level 与安全声明说明

**问题**：LLM 可能不知道 trust_level 的取值范围和含义，也不清楚 7 个安全标志的必要性。可能输出错误的 trust_level 或遗漏安全标志。

**修复**：紧随 C1 之后增加 **"TRUST LEVEL & SAFETY (mandatory):"** 章节：

- `trust_level` 必须为 `concept_geometry` 或 `reference_geometry`
- 列出 7 个安全标志及其必须为 true 的要求：
  - `non_flight_reference_only`
  - `not_airworthy`
  - `not_certified`
  - `not_for_manufacturing`
  - `not_for_installation`
  - `no_structural_validation`
  - `no_life_prediction`

**文件**：[prompt_builders.py](file:///e:/auto_detection_process/integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/authoring/prompt_builders.py#L159-L169)

**意义**：确保 LLM 输出符合安全规范，减少安全验证失败。

### C4：TURBINE_DISC_REFERENCE_EXAMPLE 修正

**问题**：`TURBINE_DISC_REFERENCE_EXAMPLE` 的 profile_stations 包含 r=1.0 的填充段（`hub_lower_fill`），这种模式会：
1. 被 A009 拦截（r=1.0 > 0.5 阈值，实际上不会被 A009 拦截，但它给 LLM 做了错误示范）
2. 让 LLM 学会"用极小半径填充段连接不同区域"的错误模式
3. 产生接近 degenerate 的几何

此外，notes 中缺少对 A009 和 A005 的明确说明。

**修复**：
1. **profile_stations 从 5 段改为 3 段 H-R-H 模式**：
   - `hub_front` (r=32.0, z=0~5)
   - `rim_outer` (r=100.0, z=5~35)
   - `hub_rear` (r=32.0, z=35~40)
   - 移除 r=1.0 的 `hub_lower_fill` 和 `hub_upper_fill`
2. **hub r=32mm 的设计考量**：
   - bore_radius=30mm，hub r=32mm，差值 2mm
   - 大于 A005 MARGIN=1.0mm，不触发 bore-vs-profile 共面检测
   - 远大于 A009 阈值 0.5mm，不会触发最小半径校验
3. **notes 从若干条扩展为 8 条**，新增关键说明：
   - Web 是 hub 与 rim 之间的过渡，不是独立的 groove
   - 不要用 cut_annular_groove 创建 web 凹陷（会产生不真实的空腔）
   - 不要使用零宽度或 Z 重叠的 profile_stations
   - **不要使用 r_mm < 0.5mm 的填充站点**（A009 明确禁止）
   - hub r=32mm 略大于 bore_radius=30mm，以避免 A005 共面检测
   - fir-tree slot 的 half_width 必须交替宽/窄以形成 undercut
   - slot 深度非递减
