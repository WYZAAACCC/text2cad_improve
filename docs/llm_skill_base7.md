# SeekFlow Engineering Tools — Generative CAD v0.5 Hard-Gate 修复规格书

目标仓库：`WYZAAACCC/seekflow-engineering`

目标目录：

```text
integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad
```

执行对象：Claude Code

目标：把当前 Generative CAD 从“局部 v0.4 scaffold”修复为真正可验证、fail-closed、metadata 可作为证明、可安全导入 SolidWorks/NX 的 LLM-Skill-Base / Generative CAD-IR 编译链路。

---

# 0. 总结判断

当前实现方向正确，但还没有达到可信编译链路。

当前最严重问题不是缺少文件，而是“关键 hard-gate 存在但未接入或可绕过”：

```text
1. validation/pipeline.py 仍然 lazy import dialect_semantics / geometry_preflight，并静默跳过 ImportError。
2. ValidationBundle 已存在，但 builder.py 没使用。
3. runtime/postconditions.py 已存在，但 pipeline/run.py 没调用。
4. builder.py 仍把 geometry_preflight 写成空 dict。
5. metadata validator 仍不要求 validation.*.ok == true。
6. import gate 不能作为真正 native import proof。
7. tools.py 没有 generative_cad_import_artifact_to_solidworks / generative_cad_import_artifact_to_nx。
8. builder.py 仍自动适配 legacy v0.1 GenerativeCADSpec。
9. repair/patch.py 仍是 scaffold，没有 path parser、patch validator、apply patch。
10. legacy v0.1 模块仍在 generative_cad 根目录，与 v0.2/v0.3/v0.4 新链路并存。
```

本次修复的目标不是继续扩展 dialect，也不是新增 CAD operation，而是把当前链路做成真正的 compiler hard-gate。

# 1. 最终目标架构

Generative CAD 正确架构必须是：

```text
User Natural Language
  ↓
Level-1 Domain Routing Skill
  ↓
DialectSelectionPlan
  ↓
Load selected Dialect Contracts + Level-2 Dialect Usage Skills
  ↓
LLM emits RawGcadDocument JSON only
  ↓
Raw validation:
    structure
    registry
    params
    ownership
    graph
    typecheck
    phase
    composition
    safety
  ↓
Canonical lowering:
    RawGcadDocument → CanonicalGcadDocument
  ↓
Canonical validation:
    dialect_semantics
    geometry_preflight
  ↓
Fixed canonical runner:
    run_canonical_gcad
  ↓
Runtime execution:
    BaseDialect.run_component
    OperationSpec.handler
    RuntimeObjectStore typed handles
  ↓
Runtime postconditions:
    final handle exists
    final handle type is solid
    all required component outputs bound
  ↓
STEP export
  ↓
Strict STEP inspection
  ↓
Generative metadata v2.1 with validation proof
  ↓
CanonicalStepArtifact
  ↓
Optional native CAD import:
    generative_cad_import_artifact_to_solidworks
    generative_cad_import_artifact_to_nx
```

Deterministic Primitive Path 和 Generative Path 只能在这里合流：

```text
validated STEP artifact + validated metadata
```

绝不能在以下层级合流：

```text
CADPartSpec
Primitive registry
Primitive compiler
geometry_primitives
SolidWorks COM feature generation
NXOpen feature generation
APDL
CadQuery source code generated by LLM
```

# 2. 不可违反的硬约束

Claude Code 必须遵守以下硬约束。

## 2.1 LLM 输出边界

LLM 只允许输出：

```text
DialectSelectionPlan JSON
RawGcadDocument JSON
RepairPatchV2 JSON
```

LLM 不允许输出：

```text
Python
CadQuery code
SolidWorks COM code
NXOpen code
APDL
shell command
file path
subprocess call
import/export call
metadata override
validation override
contract override
```

## 2.2 Runtime 输入边界

Runtime 只允许接收：

```text
CanonicalGcadDocument
```

禁止：

```text
RawGcadDocument directly into runner
dict directly into dialect handler
legacy GenerativeCADSpec into production builder
LLM raw JSON directly into runtime
```

## 2.3 SolidWorks/NX 边界

SolidWorks/NX generative import 只能接受：

```text
STEP file + valid generative_metadata_v2.1
```

禁止：

```text
generative path direct native feature-tree generation
native_rebuild_allowed == true
importing STEP without metadata validation
importing STEP with failed inspection_validation
importing STEP with failed geometry_preflight
importing STEP with failed runtime_postconditions
```

## 2.4 Primitive 隔离

不得修改以下语义：

```text
seekflow_engineering_tools/ir/cad.py
seekflow_engineering_tools/cadquery_backend/primitive_compiler.py
seekflow_engineering_tools/geometry_primitives/
PRIMITIVE_COMPILERS
PRIMITIVE_REGISTRY
CADPartSpec existing semantics
engineering_build_cad_model deterministic primitive route
```

Generative dialect 不得进入 primitive registry。

```python
RAW_STAGES = [

CANONICAL_STAGES = [
    ("dialect_semantics", validate_dialect_semantics),
```

成功时必须返回：

```python
ValidationReport(
```

```python
stages_run: list[str] = Field(default_factory=list)
```

# 4. Phase 2 — ValidationBundle 必须接入 builder

## 文件

```text
generative_cad/builder.py
```

## 当前问题

当前 builder 使用：

```python
canonical, report = validate_and_canonicalize(spec)
```

并且 metadata 中写：

```python
"geometry_preflight": {}
```

这会丢失真实 dialect_semantics / geometry_preflight report。

## 必须修改

改为：

```python
canonical, report, validation_bundle = validate_and_canonicalize_with_bundle(spec)
```

失败时返回：

```python
metrics={
    "validation": report.model_dump(),
    "validation_bundle": validation_bundle.model_dump(),
}
```

成功后，builder 需要把 `validation_bundle.to_metadata_dict()` 写入 metadata。

metadata 组装顺序必须是：

```text
1. builder validates raw → canonical and gets validation_bundle
2. builder writes canonical graph
3. runner executes canonical graph and writes initial metadata
4. builder loads metadata
5. builder reads runtime_postconditions from runner metadata
6. builder inspects STEP
7. builder overwrites metadata["validation"] with:
   - validation_bundle.to_metadata_dict()
   - runtime_postconditions
   - inspection_validation
8. builder revalidates metadata with require_validation_ok=True
9. only then returns ok=True
```

## 必须删除

生产 builder 不得自动适配 legacy spec。

删除：

```python
if hasattr(spec, 'feature_graph') and not hasattr(spec, 'components'):
    from seekflow_engineering_tools.generative_cad.compatibility.legacy_spec_adapter import adapt_legacy_spec
    spec = adapt_legacy_spec(spec)
```

替换为：

```python
if hasattr(spec, "feature_graph") and not hasattr(spec, "components"):
    return EngineeringActionResult(
        ok=False,
        software="cadquery",
        action="build_generative_cad",
        error=(
            "Legacy GenerativeCADSpec v0.1 is not accepted by the production "
            "Generative CAD builder. Convert explicitly with "
            "generative_cad.compatibility.legacy_spec_adapter in a legacy-only workflow."
        ),
    ).model_dump()
```

## 路径安全修复

当前 `graph_out` / `script_out` 不能直接 `Path(...)`。

必须改成：

```python
if graph_out is not None:
    graph_path = ensure_inside_workspace(workspace, graph_out)
else:
    graph_path = graph_dir / f"gcad_{uuid.uuid4().hex[:12]}.json"

if script_out is not None:
    script_path = ensure_inside_workspace(workspace, script_out)
else:
    script_path = script_dir / f"run_gcad_{uuid.uuid4().hex[:12]}.py"
```

## SolidWorks wrapper

```python
@tool(
    name="generative_cad_import_artifact_to_solidworks",
    description=(
        "Import a validated Generative CAD canonical STEP artifact into SolidWorks "
        "as SLDPRT. Requires generative_metadata_v2.1 import gate to pass. "
        "Does not rebuild native feature tree."
    ),
    cache=False,
    sanitize=True,
    trusted=False,
)
def generative_cad_import_artifact_to_solidworks(
    step_path: str,
    metadata_path: str,
    out_sldprt: str,
) -> dict:
    ...
```

流程：

```text
1. ensure step_path inside workspace
2. ensure metadata_path inside workspace
3. ensure out_sldprt inside workspace
4. ensure extensions: .step/.stp, .json, .sldprt
5. run validate_generative_step_artifact_for_native_import(...)
6. if gate fails, return ok=False and DO NOT call SolidWorks
7. if gate passes, call lower-level SolidWorks STEP import helper
8. return metrics including native_rebuild_allowed=False
```

如果当前 SolidWorks tool 没有 helper，抽取：

```text
solidworks/importers.py
  import_step_as_sldprt(config, input_step, out_sldprt) -> dict
```

不要直接调用 decorated tool function。

## 必须增加 model_validator

```python
@model_validator(mode="after")
def validate_route_invariants(self):
    if self.route_decision == "generative_cad_ir" and not self.selected_dialects:
        raise ValueError("generative_cad_ir requires selected_dialects")

    if self.route_decision == "deterministic_primitive" and self.selected_dialects:
        raise ValueError("deterministic_primitive must not select generative dialects")

    if self.route_decision == "unsupported" and not self.unsupported_capabilities:
        raise ValueError("unsupported requires unsupported_capabilities")

    seen = set()
    for item in self.selected_dialects:
        if item.dialect in seen:
            raise ValueError(f"duplicate selected dialect: {item.dialect}")
        seen.add(item.dialect)

    return self
```

## 添加 catalog 校验函数

```python
def validate_selection_plan_against_catalog(
    plan: DialectSelectionPlan,
    catalog: dict,
) -> tuple[bool, list[dict]]:
    allowed = {d["dialect_id"] for d in catalog.get("dialects", [])}
    issues = []

    for item in plan.selected_dialects:
        if item.dialect not in allowed:
            issues.append({
                "code": "unknown_selected_dialect",
                "message": f"selected dialect {item.dialect!r} is not in catalog",
            })

    return not issues, issues
```
