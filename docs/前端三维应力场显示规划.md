# 前端三维应力场显示规划

> 目标：把 `fea3d` 流水线算出的**三维应力场**（26,479 节点 × 6 应力分量，1/120 循环对称扇区）在浏览器中以颜色云图显示。
> 前置事实（已实测）：全盘 STL 8.7MB ≈ 174k 三角形 / **522k 非索引顶点**；应力场 CSV 3.3MB；现有着色代码是 O(V×N) 线性扫描，3D 数据下不可用。
> 编写日期：2026-07-16。

---

## 1. 方案选型：为什么是"双视图"而不是体渲染

| 候选方案 | 结论 | 理由 |
|---|---|---|
| GPU 体渲染 (raymarching 3D texture) | ❌ | 需把非结构化四面体场重采样成 3D 纹理；集成复杂、移动端 GPU 吃力；对"看表面+看剖面"的工程需求过度设计 |
| 点云直显 26k 节点 | ❌ | 不直观，没有连续云图观感 |
| Marching cubes 等值面 | ❌ | 实现重，且工程上更常用的是云图而非等值面 |
| **视图 A：全盘表面云图**（对称映射 + 顶点着色） | ✅ 主视图 | 复用已加载的 STL，一次性计算顶点色，零每帧开销 |
| **视图 B：扇区剖视云图**（FE 表面网格直传） | ✅ 核心创新点 | 扇区的三个切割面（θ=3°、θ=9°、z=0）**天然就是剖面**——z=0 面是完整的 R-Z 截面，直接暴露**内部**应力分布。这才是"真三维应力场"，不加任何 GPU 剖切技巧 |
| 动态剖切平面 (clippingPlanes) | 🔶 阶段 2 | Three.js 原生支持，但 STL 是空壳，剖切后需要"补盖"（capping）才有意义，复杂度高收益低 |

**关键洞察**：表面云图只能看到表面应力；而循环对称扇区模型的切割面把内部应力"免费"暴露出来了——z=0 对称面就是过盘心的完整子午剖面（bore→hub→web→rim 的内部应力一目了然），θ 切割面是穿过榫槽的周向剖面。因此**视图 B 不需要任何额外计算就是内部三维应力场的剖视图**。

## 2. 总体数据流

```
post3d.py (Python, 新增导出)
  ├─ stress_field_3d.bin   全部 26,479 节点: pos(x,y,z) + 6分量 + SF  → 视图A查询用
  └─ sector_surface.json   FE扇区表面网格: positions/indices/每顶点6分量+SF → 视图B直渲
        │  (新增 GET /api/fea3d/files/{job}/{filename})
        ▼
前端 Web Worker (新文件 stressWorker.ts)
  ├─ 解析二进制 → 构建均匀空间哈希网格 (一次, ~26k 点)
  ├─ 遍历 STL 522k 顶点: 对称映射(x,y,z)→扇区(r,θ',z') → O(1) 最近邻 → jet 颜色
  └─ postMessage(Float32Array colors, [transfer])   ← 主线程零拷贝接收
        ▼
Viewport3D (改造)
  ├─ 视图A: STL geometry.setAttribute('color', ...)   静态顶点色, 无每帧开销
  ├─ 视图B: sector_surface → THREE.BufferGeometry (索引网格+顶点色)
  ├─ HTML 覆盖层: 颜色条图例 (vmin/vmax/单位/屈服线刻度)
  └─ 工具栏: 分量切换 (σvm/σr/σθ/σz/SF) · 视图A/B切换 · 应力开关(已有)
```

## 3. 数据格式规格（后端新增导出，post3d.py 内实现）

### 3.1 `stress_field_3d.bin`（视图 A 查询用，~1.0MB）

纯二进制小端 Float32，避免 3.3MB CSV 传输与 JSON 解析开销：

```
Header (16 bytes): uint32 magic=0x33444653 ("SFD3") | uint32 nodeCount | uint32 fieldCount=8 | uint32 reserved
Body: nodeCount × 8 × float32, 每节点连续: [x, y, z, s_vm, s_r, s_hoop, s_axial, sf]
```

26,479 × 8 × 4B ≈ 847KB。前端 `DataView`/`Float32Array` 直接切片，无解析成本。

### 3.2 `sector_surface.json`（视图 B 直渲，~1-2MB，gzip 后更小）

Python 侧从 tet10 网格提取**边界三角面**（出现次数=1 的四面体面；只用角节点，因 SecondOrderLinear=1 网格本就是直边，几何无损）：

```json
{
  "meta": {"job": "...", "sector_deg": 6, "theta_low_deg": 3, "z_half": 38,
            "vmin": {"s_vm": ..}, "vmax": {"s_vm": ..}},
  "positions": [x0,y0,z0, x1,y1,z1, ...],      // 仅表面节点, 重新紧凑编号
  "indices":   [i0,i1,i2, ...],                 // 三角索引
  "fields": {"s_vm": [...], "s_r": [...], "s_hoop": [...], "s_axial": [...], "sf": [...]}
}
```

预估表面节点 ~8k、三角形 ~16k——对 Three.js 是轻量级。**注意法向**：边界面提取时按"第四顶点在面外侧"定向，保证外法向一致，否则光照会花。

### 3.3 服务端路由（main.py 新增，约 10 行）

```
GET /api/fea3d/jobs                     → 列出 output/fea3d_jobs/ 下含 metrics.json 的作业
GET /api/fea3d/files/{job}/{filename}   → 白名单文件服务 (metrics.json / stress_field_3d.bin /
                                           sector_surface.json / result_summary.txt)
```

`job`/`filename` 均须过 `[A-Za-z0-9_\-\.]+` 白名单（现有 `/api/files` 已被审查报告点名路径遍历，新路由不要重蹈覆辙）。

## 4. 视图 A：全盘表面云图（核心算法）

### 4.1 对称映射（最易错，公式固定如下）

扇区覆盖 θ∈[3°,9°]、z∈[0,38]，全盘任意 STL 顶点 (x,y,z) 映射到扇区坐标：

```ts
const rad2deg = 180 / Math.PI;
const r = Math.hypot(x, y);
let theta = Math.atan2(y, x) * rad2deg;          // (-180, 180]
let t = (theta - 3) % 6;                          // 周期 6°
if (t < 0) t += 6;                                // JS 的 % 对负数返回负值! 必须修正
const thetaSector = 3 + t;                        // ∈ [3, 9)
const zSector = Math.abs(z);                      // 镜像对称
// 查询点 (直角坐标, 在扇区参考系):
const qx = r * Math.cos(thetaSector / rad2deg);
const qy = r * Math.sin(thetaSector / rad2deg);
const qz = zSector;
```

### 4.2 空间哈希最近邻（替代线性扫描）

- 构建：把 26,479 个扇区节点按 `cell = floor(pos / h)` 装入 `Map<cellKey, int[]>`，`h = 4mm`（≈网格平均尺寸的一半到一倍之间）；
- 查询：查 3×3×3 邻域桶取最近点；空则扩环（最多 3 环，再空则返回 NaN→灰色）；
- 复杂度：522k 顶点 × ~均摊十几个候选点 ≈ 10⁷ 级运算，Worker 内 <1 秒。
- **必须在 Web Worker 中执行**，颜色 Float32Array 用 transferable 传回主线程（零拷贝）。主线程只做 `geometry.setAttribute('color', ...)`。

### 4.3 缓存与切换

Worker 结果按 `(jobId, 分量名)` 缓存于内存（每份 6.3MB 颜色缓冲，最多缓存 5 分量 ≈ 32MB，可接受；LRU 上限 5）。切换分量时若命中缓存则即时切换，否则 Worker 重算（仅重跑颜色映射部分，最近邻索引结果 `Uint32Array(522k)≈2MB` 首次算好后复用——**这是最重要的性能优化：几何→节点的映射只算一次，换分量只是查表+调色**）。

## 5. 视图 B：扇区剖视云图

- `sector_surface.json` → `THREE.BufferGeometry`（indexed，`position` + `color`），`MeshStandardMaterial({vertexColors:true, side:THREE.DoubleSide})`；
- 默认摆放：扇区放大置于原点或与全盘同位（全盘 STL 切换为 `opacity:0.08` 幽灵模式，突出扇区）；提供"仅扇区/扇区+幽灵全盘"切换；
- 切割面即剖面：z=0 面朝上摆放（或提供预设相机"剖面视角"按钮），用户一眼看到内部 R-Z 应力分布；
- 分量切换直接改 color attribute（表面节点仅 ~8k，主线程重算颜色 <5ms，无需 Worker）。

## 7. 性能预算与降级策略

| 项 | 预算 | 实测依据/降级 |
|---|---|---|
| bin 下载 | 847KB，<1s 局域网 | gzip 后 ~500KB |
| sector_surface | ~1-2MB | 同上 |
| Worker 建索引 | <1.5s 一次性 | 期间视图正常显示未着色模型 + 工具栏 spinner，绝不阻塞主线程 |
| 分量切换（已建索引） | <100ms | 查表+调色 O(V) |
| 每帧渲染开销 | **0 增量** | 静态顶点色，无 shader 定制、无每帧计算 |
| 内存 | STL几何 ~25MB + 颜色缓存 ≤32MB + 映射表 2MB | 移动端/低配：`navigator.hardwareConcurrency<4` 或 STL 顶点 >1M 时自动只启用视图 B（扇区仅 ~8k 节点，任何设备都流畅），视图 A 按钮置灰并提示 |
| 颜色范围 | 默认全局 vmin/vmax（来自 meta） | 提供"钳位到屈服"开关：vmax=σy(650°C)=950，超出显红，工程上更实用 |

## 8. 易错点清单（实现时对照）

1. **JS `%` 负数陷阱**：`(theta-3)%6` 对 θ<3° 返回负值，必须 `if(t<0)t+=6`（§4.1 已写死）。
2. **STL 非索引**：STLLoader 输出非索引几何，522k 顶点中同一空间点出现 ~3 次，直接逐顶点算即可，**不要**先做 mergeVertices（400ms+ 且破坏法向）。
3. **坐标系一致性**：STL 顶点坐标在 `geometry.translate(центering)` 之前才是物理 mm 坐标——现有代码 [Viewport3D.tsx:60-71](../app/text-to-cad/src/components/Viewport3D.tsx#L60-L71) 先居中再着色会引入平移误差！本盘 bbox 本就近似关于原点对称（x,y∈±250, z∈±38），居中平移量≈0，但**必须改为：先记录平移量，映射时把顶点坐标加回去**，否则换一个不对称零件就错。
4. **jet 公式同源**：现有 Viewport3D 和 StressHeatmap 各有一份 jet 拷贝，加 ColorbarOverlay 就是第三份——必须抽 `src/colormap.ts` 单一来源，否则图例颜色与模型颜色对不上。
5. **Worker 与 R3F**：颜色缓冲回来后 `needsUpdate=true` 即可，不要重建 geometry（会闪烁）。
6. **法向定向**（视图 B，Python 侧）：边界三角面用"tet 第四顶点在内侧"规则统一外法向；否则用 DoubleSide + flatShading 兜底也可接受。
7. **NaN 处理**：映射失败（空桶 3 环）着灰色 `#666`，并统计比例——若 >1% 说明对称映射公式或单位错了，console.warn。
8. **量纲**：bin 中坐标是 mm，STL 也是 mm，无缩放；若未来前端对模型做 scale，映射必须用原始几何坐标。

## 9. 实施步骤（建议顺序）与验收

1. **post3d.py 导出 bin + sector_surface**（含表面提取与法向定向）→ 验收：对 `_test_prepare` 作业重跑 post，两文件生成，节点数/三角数打印合理；
2. **main.py 新增 /api/fea3d 路由** → 验收：curl 能取回文件，非法路径 404；
3. **colormap.ts 抽取 + ColorbarOverlay** → 验收：图例渐变与 2D 热图一致；
4. **stressWorker.ts + 视图 A** → 验收：打开应力着色后 <2s 出全盘云图，交互旋转 60fps，bore 附近红色（maxVM 区），slot/rim 蓝绿色，灰色顶点比例 <1%；
5. **视图 B 扇区剖视** → 验收：z=0 剖面可见内部应力梯度（bore 红 → rim 蓝），θ 切割面穿过榫槽；
6. **FeaTab 集成** → 验收：选作业 → 显示 metrics → 一键在 3D 视图中显示；分量切换 σvm/σθ 云图明显不同（σθ 在 bore 更集中）。

**总验收标准**：涡轮盘 15,000rpm 工况下，视图 A 全盘云图 + 视图 B 剖视图的颜色分布与 `metrics.json` 的分区指标定性一致（bore≈1049 红、hub≈905 橙、web≈569 绿、rim≈416 蓝），且旋转/缩放全程流畅（≥50fps），分量切换 <100ms。

## 10. 明确不做（本期范围外）

- 体渲染 / 等值面 / 动态剖切补盖（阶段 2 再评估）；
- 悬停探针读数（raycast → 最近节点 → tooltip，阶段 2 加，依赖已有映射表，成本低）;
- 变形动画（需导出位移场，APDL 加 3 列 `*VGET,U,X/Y/Z` 即可，接口预留）；
- 扇区 ×120 复制成整盘 FE 网格（522k 节点 ×120 无意义，视图 A 已解决全盘观感）。
