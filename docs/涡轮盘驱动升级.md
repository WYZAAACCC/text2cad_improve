你这个判断是对的：**如果给涡轮盘单独写一个 `axisymmetric_turbine_disk` primitive，那确实没有研究价值。** 这会退化成传统参数化模板库，LLM 只是填参数，不能证明它真的会建模。

正确处理方式应该是：**不要增加“涡轮盘专用 primitive”，而是增加/完善一组足够表达工业 CAD 建模过程的通用操作，让 LLM 用这些通用操作组合出涡轮盘。**

换句话说，目标不是：

```text
用户说涡轮盘
→ 调 turbine_disk_primitive(...)
```

而应该是：

```text
用户说涡轮盘
→ LLM 理解结构语义
→ 分解成通用 CAD 特征序列
→ sketch/revolve/cut/extrude/pattern/boolean/fillet
→ 系统验证这些通用操作组合出来的结果是否满足涡轮盘语义
```

---

## 1. 现在的问题不是“缺少涡轮盘 primitive”，而是“通用操作空间不够”

当前 `test_v11` 的 route plan 已经识别出对象是 `axisymmetric_turbine_disk`，但直接选择了 `deterministic_primitive`，同时又在 `unsupported_capabilities` 里承认当前 primitive 不能表达多级 fir-tree 槽和特定密封槽区域。这说明路由器明知表达力不足仍然放行了。([GitHub][1])

`llm_raw.json` 里，LLM 生成的是：

```text
revolve_profile
→ cut_center_bore
→ cut_circular_hole_pattern
→ cut_rim_slot_pattern
→ cut_annular_groove
→ apply_safe_chamfer
```

这看起来是通用流程，但实际上不是。因为 `revolve_profile` 和 `cut_rim_slot_pattern` 的参数空间太窄，导致 LLM 无法正确表达真实涡轮盘剖面和枞树槽。([GitHub][2])

具体来说，当前 `RevolveProfileParams` 只有：

```text
r_mm
z_front_mm
z_rear_mm
```

它更像是在描述一组轴向区间，而不是一个真正的有序 R-Z 闭合草图。([GitHub][3]) 真实涡轮盘应该是“轴向厚度随半径变化”，即 `z_front(r)` / `z_rear(r)`，需要有序闭合剖面，而不是让系统按 Z 排序生成外轮廓。当前 handler 里多 station 的 revolve 会把点按 Z 排序，这会天然破坏 hub/web/rim 这种径向剖面的拓扑顺序。([GitHub][4])

同理，当前 `cut_rim_slot_pattern` 只有 `depth_mm + half_width_mm` 的 station 折线，最多只能做“外缘周期折线槽”，不能表达真正的枞树槽的齿肩、颈部、工作面、非工作面、圆角和底部槽根。([GitHub][3])

所以核心不是要加涡轮盘 primitive，而是要补齐这些通用能力：

```text
1. 任意有序 2D 草图 sketch
2. 闭合草图旋转 revolve
3. 草图拉伸 extrude
4. 草图切除 extrude_cut
5. 布尔 subtract / union / intersect
6. 极阵列 polar pattern
7. 局部坐标系 / tangent frame
8. 线、圆弧、样条、倒圆、约束
9. 基于 tag 的边选择和倒角/圆角
10. 几何后验测量与语义验证
```

有了这些，LLM 才能真正“建模”，而不是调用模板。

---

## 2. 涡轮盘应该被拆成通用操作链，而不是专用 primitive

正确的通用建模链应该是这样：

```text
Step 1：建立 R-Z 剖面草图
Step 2：绕 Z 轴旋转，得到 hub/web/rim 基础盘体
Step 3：建立中心孔，或者在 R-Z 草图里直接包含 bore
Step 4：在外缘局部切向坐标系中画一个 fir-tree 槽截面草图
Step 5：把该槽草图沿 Z 轴拉伸成 cutter
Step 6：对 cutter 做 polar pattern
Step 7：用 patterned cutters 从 rim 上 boolean subtract
Step 8：添加端面环槽、密封槽、倒角、圆角
Step 9：做 closed solid / valid solid / body count / bbox / 截面语义检查
```

这套流程里没有任何 `turbine_disk()` 或 `fir_tree_slot()` 专用函数。所谓“枞树槽”只是一个**带语义约束的通用 2D 草图 + 拉伸切除 + 圆周阵列**。

## 3. 应该新增的是“通用 sketch/feature IR”，不是“领域模板”

你现在系统里最应该新增的不是：

```text
axisymmetric_turbine_disk
fir_tree_slot
blade_root_slot
```

而是下面这种通用操作。

### 3.1 通用闭合草图旋转

```json
{
  "op": "sketch_revolve",
  "params": {
    "plane": "XZ",
    "axis": "Z",
    "sketch": {
      "units": "mm",
      "closed": true,
      "curves": [
        {"type": "line", "from": [60, -38], "to": [120, -38]},
        {"type": "line", "from": [120, -38], "to": [170, -22]},
        {"type": "line", "from": [170, -22], "to": [215, -16]},
        {"type": "line", "from": [215, -16], "to": [250, -32]},
        {"type": "line", "from": [250, -32], "to": [250, 32]},
        {"type": "line", "from": [250, 32], "to": [215, 16]},
        {"type": "line", "from": [215, 16], "to": [170, 22]},
        {"type": "line", "from": [170, 22], "to": [120, 38]},
        {"type": "line", "from": [120, 38], "to": [60, 38]},
        {"type": "line", "from": [60, 38], "to": [60, -38]}
      ]
    }
  }
}
```

这里的 `[R, Z]` 是有序闭合剖面点。这个操作是通用的，可以建涡轮盘，也可以建法兰、轴套、喷嘴、泵轮、壳体、阀体、端盖。

### 3.2 通用局部草图切除

外缘枞树槽不要做专用 op，而是这样：

```json
{
  "op": "create_sketch",
  "params": {
    "frame": {
      "origin": [250, 0, 0],
      "x_axis": "radial_inward",
      "y_axis": "tangential",
      "z_axis": "axial"
    },
    "sketch": {
      "closed": true,
      "curves": [
        {"type": "line", "from": [0, 3.5], "to": [4, 8]},
        {"type": "arc", "from": [4, 8], "to": [7, 5], "radius": 1.0},
        {"type": "line", "from": [7, 5], "to": [11, 7]},
        {"type": "arc", "from": [11, 7], "to": [15, 4], "radius": 1.0},
        {"type": "line", "from": [15, 4], "to": [21, 3.5]},
        {"type": "arc", "from": [21, 3.5], "to": [21, -3.5], "radius": 2.0},
        {"type": "line", "from": [21, -3.5], "to": [15, -4]},
        {"type": "arc", "from": [15, -4], "to": [11, -7], "radius": 1.0},
        {"type": "line", "from": [11, -7], "to": [7, -5]},
        {"type": "arc", "from": [7, -5], "to": [4, -8], "radius": 1.0},
        {"type": "line", "from": [4, -8], "to": [0, -3.5]},
        {"type": "line", "from": [0, -3.5], "to": [0, 3.5]}
      ]
    }
  }
}
```

然后：

```json
{
  "op": "extrude",
  "params": {
    "input_sketch": "slot_sketch",
    "direction": "Z",
    "distance": 80,
    "both": true
  }
}
```

再：

```json
{
  "op": "polar_pattern",
  "params": {
    "input_body": "slot_cutter",
    "axis": "Z",
    "count": 60,
    "angle_deg": 360
  }
}
```

最后：

```json
{
  "op": "boolean_subtract",
  "params": {
    "target": "disk_body",
    "tools": "slot_cutters_pattern"
  }
}
```

这才是你要的：**LLM 用通用操作构建正确涡轮盘**。

## 4. prompt 要改成“强制通用操作分解”，而不是描述形状后让它自由选择

你的 prompt 不能只说“建一个涡轮盘，要有枞树槽”。这样 LLM 很容易选择最近似的已有 primitive 或偷懒用 `cut_rim_slot_pattern`。

应该在 prompt 里明确限制：

```text
禁止使用任何 turbine_disk、axisymmetric_turbine_disk、fir_tree_slot、blade_root_slot 等领域专用 primitive。
必须只使用通用 CAD 操作完成：
1. sketch_revolve / revolve closed sketch
2. create_sketch
3. extrude
4. boolean_subtract
5. polar_pattern
6. fillet / chamfer
7. measurement / validation

如果无法用这些通用操作表达，就 fail-closed，不能退化成简化齿轮盘、普通外缘槽或阶梯圆盘。
```

一份更适合你系统的 prompt 可以这样写：

```text
请用通用 CAD 建模操作生成一个航空发动机高压涡轮盘 reference geometry，单位 mm，仅用于非飞行、非认证、非制造的概念几何验证。

严格禁止使用 turbine_disk、axisymmetric_turbine_disk、fir_tree_slot、blade_root_slot 等领域专用 primitive。必须仅通过通用操作完成建模，包括：
- 有序闭合 2D 草图；
- revolve；
- extrude；
- boolean subtract；
- polar pattern；
- fillet/chamfer；
- 几何测量与验证。

建模步骤必须如下：

1. 在 XZ 平面绘制有序闭合 R-Z 剖面草图，R 为半径，Z 为轴向。该剖面必须体现：
   - 中心孔半径约 60 mm；
   - hub 区域 R=60–120 mm，轴向厚度最大，约 70–80 mm；
   - web 区域 R=120–215 mm，厚度随半径逐步减小，外侧约 28–35 mm；
   - rim 区域 R=215–250 mm，厚度重新增大到约 55–65 mm；
   - hub/web/rim 之间用斜线或圆弧过渡，不能建成普通阶梯圆盘或塔形件。

2. 将该闭合剖面绕 Z 轴 revolve 360°，生成单一基础盘体。

3. 在外缘 R≈250 mm 处建立局部切向坐标系：
   - local X 指向径向向内；
   - local Y 指向圆周切向；
   - local Z 为轴向。

4. 在该局部坐标系内绘制一个雌槽 fir-tree 形状的通用闭合草图。注意：这不是调用 fir_tree primitive，而是用 line/arc/fillet 画出槽型。
   槽型必须沿径向向内呈现：
   - 外口窄颈；
   - 第一宽齿肩；
   - 中间窄颈；
   - 第二宽齿肩；
   - 下部窄颈；
   - 圆角槽根。
   左右两侧镜像对称，所有齿根和转角必须有圆角或圆弧近似，不能是普通矩形槽、三角槽或锯齿槽。

5. 将该槽草图沿 Z 轴双向拉伸，生成贯穿 rim 厚度的 cutter body。

6. 对 cutter body 绕 Z 轴做 60 等分 polar pattern。

7. 用 patterned cutters 对基础盘体执行 boolean subtract，形成 60 个轴向贯通的枞树形轮槽。

8. 添加必要的小圆角/倒角，但不能破坏槽型、rim posts 或 hub/web/rim 主剖面。

9. 最终必须验证：
   - body_count = 1；
   - closed solid = true；
   - valid solid = true；
   - 外形剖面满足 hub 厚、web 薄、rim 加厚；
   - fir-tree 槽具有至少两个宽齿肩和两个窄颈；
   - 不允许退化为普通齿轮、普通开槽圆盘或阶梯圆盘。

如果当前操作集不能创建 ordered closed sketch、extrude cutter、polar pattern 或 boolean subtract，则必须 fail-closed，并报告缺失的通用 CAD 操作，不得调用领域专用 primitive 替代。
```

## 5. 系统层面应该这样改

### 第一，禁用涡轮盘专用路由

只要任务是验证 LLM 通用建模能力，route plan 里就不应该出现：

```json
"route_decision": "deterministic_primitive",
"selected_primitive": "axisymmetric_turbine_disk"
```

而应该是：

```json
"route_decision": "general_feature_modeling",
"selected_primitive": null,
"allowed_ops": [
  "create_sketch",
  "sketch_revolve",
  "extrude",
  "boolean_subtract",
  "polar_pattern",
  "fillet",
  "chamfer",
  "measure_section"
]
```

领域知识可以参与规划，但不能隐藏成 primitive。也就是说，可以有：

```text
domain_skill: turbomachinery_shape_constraints
```

但不能有：

```text
primitive: turbine_disk
```

### 第二，把“领域知识”变成 verifier，而不是 generator

涡轮盘知识不要用来直接生成模型，而要用来检查模型是否对。

例如 verifier 检查：

```text
1. 是否存在 revolve 生成的基础盘体；
2. R-Z 剖面是否是 hub 厚、web 薄、rim 加厚；
3. 是否存在外缘轴向贯通切槽；
4. 切槽是否由 sketch + extrude + polar pattern + boolean subtract 生成；
5. slot profile 是否出现 neck/lobe/neck/lobe/neck 的宽度交替；
6. slot 数量是否合理；
7. rim posts 是否保留；
8. 最终 solid 是否 closed 和 valid。
```

这样才能判断 LLM 是否真的构建正确，而不是调用模板蒙混过关。

### 第三，修正最终几何后验检查

现在 `output.metadata.json` 里已经出现：

```text
closed = false
is_valid_solid = false
```

但 `geometry_postcheck.ok` 仍然是 true。这个必须改。只要约束里写了 `require_closed_solid=true`，最终 `closed=false` 或 `is_valid_solid=false` 就必须构建失败，不能只写进 metadata。([GitHub][5])

否则 LLM 生成一个拓扑坏掉的模型，系统仍然会告诉用户“成功”。

---

## 6. 最重要的一点：通用操作集必须足够“CAD 原子化”

你不需要涡轮盘 primitive，但你确实需要更底层、更通用、更强的 CAD 原子操作。

最低限度应该有：

```text
Sketch 层：
- create_sketch
- line
- arc
- spline / polyline
- close_profile
- mirror
- offset
- fillet_sketch_corner
- tag_curve / tag_vertex

Feature 层：
- revolve
- extrude
- sweep
- loft，可选
- cut_extrude
- boolean_subtract
- boolean_union

Pattern 层：
- linear_pattern
- polar_pattern

Reference Geometry 层：
- create_plane
- create_frame
- tangent_frame_on_cylinder
- axis
- datum plane

Topology 层：
- select_edges_by_tag
- select_edges_by_radius
- select_faces_by_normal
- fillet
- chamfer

Validation 层：
- section_cut
- measure_bbox
- measure_body_count
- measure_closed_solid
- measure_radial_thickness_profile
- measure_slot_width_depth_profile
```

这不是专用模板库，而是正常工业 CAD 的“操作语言”。LLM 的价值就在于：**它能把复杂工程对象翻译成这些通用操作序列。**

## 7. 对 `test_v11` 的正确处理策略

我建议你不要简单修 prompt 后重跑，而是把 `test_v11` 改成一个“通用操作能力测试”。

测试通过条件应该是：

```text
1. route_plan 不允许 selected_primitive = axisymmetric_turbine_disk；
2. raw/canonical graph 必须包含 sketch_revolve；
3. 基础盘体必须来自 ordered closed R-Z sketch；
4. fir-tree 槽必须来自 create_sketch + extrude + polar_pattern + boolean_subtract；
5. slot 草图宽度必须出现至少两次 neck/lobe 交替；
6. 不能使用 cut_rim_slot_pattern 这种过窄的伪通用槽操作；
7. 最终 closed=true；
8. 最终 is_valid_solid=true；
9. 截面测量满足 hub 厚、web 薄、rim 加厚；
10. 不满足则 fail-closed。
```

最终判断就是：

```text
不要做 turbine_disk primitive。
要做 general CAD operation grammar。
不要让领域知识生成模型。
要让领域知识约束和检查模型。
不要让 LLM 绕过建模过程。
要让 LLM 明确输出 sketch/revolve/extrude/pattern/boolean/fillet 的完整构建链。
```

这才符合你这个系统真正有价值的方向。
