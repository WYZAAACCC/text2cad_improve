# 涡轮盘分层建模架构修复报告

> **报告范围**：基于 `turbine_disc/test_v3/测试诊断报告_v4.md` 的深度诊断，针对 test_v3 测试中发现的 LLM 零宽度站点问题，从架构层面建立的分层建模修复方案。
> **修复目标**：建立正确的"分层建模"架构，使 LLM 能正确建模涡轮盘等复杂轴对称零件，而非在 revolve_profile 中用半径变化表达 Web 内凹。
> **修复日期**：2026-07-06

---

## 一、问题诊断

### 1.1 测试现象

test_v3 测试中，LLM 试图在 `revolve_profile` 的 `profile_stations` 中用**零宽度站点**（`z_front_mm == z_rear_mm`）表达 Rim(250)→Web(210) 的瞬时径向跳变，被 A001 校验拦截：

```
a001_z_order: z_rear_mm (42.0) must be > z_front_mm (42.0) and finite
```

LLM 的实际输出（`llm_raw.json`）：

```json
{"r_mm": 250, "z_front_mm": 42, "z_rear_mm": 42, "label": "jump"},  // 零宽度
{"r_mm": 210, "z_front_mm": 42, "z_rear_mm": 78, "label": "web"},
{"r_mm": 250, "z_front_mm": 78, "z_rear_mm": 78, "label": "jump"}   // 零宽度
```

### 1.2 根因分析

通过深度代码与架构分析，发现问题根本原因不在 A001 过于严格，而在**架构层面缺少正确的建模指导**：

#### 根因链

```
LLM 零宽度站点（A001 拦截）
    ↑
LLM 试图在 revolve_profile 中表达 Rim→Web 跳变
    ↑
LLM 学到了 TURBINE_DISC_REFERENCE_EXAMPLE 的错误模式
（示例展示"在 profile 中用半径变化表达 Web 内凹"）
    ↑
TURBINE_DISC_REFERENCE_EXAMPLE 本身是错误的建模方式
（沙漏形外轮廓，不是真正的涡轮盘）
    ↑
缺少正确的"分层建模"指导
（revolve 纯外轮廓 + cut_annular_groove 侧面切削）
    ↑
cut_annular_groove 语义限制
（无法精确指定 Z 区间，只能从端面切削 depth 毫米）
```

### 1.3 四个具体问题

| # | 问题 | 严重度 | 位置 | 根因层 |
|---|---|---|---|---|
| 1 | TURBINE_DISC_REFERENCE_EXAMPLE 示范错误建模方式 | 严重 | `base_packages/axisymmetric/package.py` | Example 错误 |
| 2 | NODE_PARAMS_SYSTEM_PROMPT 缺少 axisymmetric 规则 | 严重 | `authoring/prompt_builders.py` | Prompt 缺失 |
| 3 | cut_annular_groove 无法精确定位 Z 区间 | 严重 | `bases/axisymmetric/models.py` + `dialects/axisymmetric/handlers.py` | Handler 语义限制 |
| 4 | ProfileStation validator 与 A001 不一致 | 中等 | `bases/axisymmetric/models.py` | Schema 不一致 |

### 1.4 TURBINE_DISC_REFERENCE_EXAMPLE 的错误

**修复前**（沙漏形外轮廓）：

```python
"profile_stations": [
    {"r_mm": 100.0, "z_front_mm": 0.0,  "z_rear_mm": 5.0,  "label": "rim_top"},
    {"r_mm": 80.0,  "z_front_mm": 5.0,  "z_rear_mm": 10.0, "label": "rim_to_web"},
    {"r_mm": 80.0,  "z_front_mm": 10.0, "z_rear_mm": 30.0, "label": "web"},
    {"r_mm": 100.0, "z_front_mm": 30.0, "z_rear_mm": 35.0, "label": "web_to_rim"},
    {"r_mm": 100.0, "z_front_mm": 35.0, "z_rear_mm": 40.0, "label": "rim_bottom"},
]
```

**问题分析**：
- 这是"沙漏形"外轮廓（两端 r=100、中间 r=80）
- 不是真正的涡轮盘几何——真正的涡轮盘 Rim 外径应覆盖整个 Z=0→40
- Web 应该是 Rim 内侧的凹陷，是侧面减材特征，不应出现在外轮廓中
- LLM 学到了这种错误模式，试图在 profile 中表达 Rim→Web 跳变

### 1.5 cut_annular_groove 的语义限制

**修复前**：

```python
z_pos = bb.zmin if side == "front" else bb.zmax
extrude_dir = depth if side == "front" else -depth
```

- `side="front"` → 从 zmin 开始切削 depth 毫米
- `side="rear"` → 从 zmax 开始切削 depth 毫米
- **无法精确定位 Z 区间**（如 Web 凹陷 Z=10→30）

这导致即使 LLM 想用正确的"分层建模"思路，也无法用现有操作精确表达涡轮盘的 Web 凹陷。

## 二、修复方案

### 2.1 核心策略：分层建模

```
┌─────────────────────────────────────────────────────────────────┐
│ Layer 1: revolve_profile — 纯外轮廓                              │
│  └── r_mm = 最大半径，覆盖整个 Z 范围                            │
│      示例：r=100, Z=0→40（圆柱外轮廓）                           │
├─────────────────────────────────────────────────────────────────┤
│ Layer 2: cut_annular_groove — 侧面凹陷                          │
│  └── z_position_mm + depth_mm 精确定位 Z 区间                    │
│      示例：z_position=10, depth=20 → 切削 Z=10→30 的 Web 凹陷   │
├─────────────────────────────────────────────────────────────────┤
│ Layer 3: cut_center_bore — 中心孔                               │
│  └── diameter_mm + through_all                                  │
├─────────────────────────────────────────────────────────────────┤
│ Layer 4: cut_rim_slot_pattern — 榫槽                            │
│  └── count + slot_depth_mm + slot_profile                       │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 修复决策

| 决策 | 选择 | 理由 |
|---|---|---|
| cut_annular_groove 扩展方式 | 增加 `z_position_mm` 可选参数 | 最小化修改，向后兼容 |
| TURBINE_DISC_REFERENCE_EXAMPLE 修正方式 | 改为分层建模 | 展示正确的建模思路 |
| ProfileStation validator | 从 `>` 改为 `>=` | 与 A001 保持一致 |
| A001 是否放宽 | 不放宽 | 从 Prompt 层面引导正确建模 |
| side 参数语义 | 保持，决定切削方向 | z_position_mm 只影响起始位置 |

```python
def handle_cut_annular_groove(node: CanonicalNode, ctx: RuntimeContext) -> dict[str, str]:
    # ... 参数解析 ...
    z_position = p.get("z_position_mm")
    # ... 校验 ...
    try:
        bb = body.val().BoundingBox()
        # z_position_mm 优先；缺失时回退到 side 决定的端面
        if z_position is not None:
            z_pos = float(z_position)
        else:
            z_pos = bb.zmin if side == "front" else bb.zmax
        extrude_dir = depth if side == "front" else -depth
        ring = (
            cq.Workplane("XY").workplane(offset=z_pos)
            .circle(outer / 2.0).circle(inner / 2.0).extrude(extrude_dir)
        )
        result = body.cut(ring)
    # ... 异常处理 ...
```

### 修复 3：在 NODE_PARAMS_SYSTEM_PROMPT 添加 axisymmetric 规则

**文件**：[prompt_builders.py:124-136](file:///e:/auto_detection_process/integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/authoring/prompt_builders.py#L124-L136)

**修改内容**：在 `For shell:` 段落之后添加 axisymmetric revolve_profile 建模规则

```python
For axisymmetric revolve_profile:
- profile_stations describe the OUTER contour only — each Z position has exactly
  ONE radius (the maximum radius at that Z). Inner features (bore, web, hub) are
  created by separate cut operations, NOT by reducing r_mm in profile_stations.
- z_rear_mm must be STRICTLY GREATER than z_front_mm (zero-width stations are
  forbidden and will be rejected by validation).
- If a radius transition is needed (e.g., rim_to_hub), use a short but non-zero
  segment (>= 0.5mm) to represent the transition.
- Use cut_annular_groove with z_position_mm to machine side features at specific
  Z ranges (e.g., web recess from Z=10 to Z=30).
- DO NOT model an hourglass shape (narrow middle, wide ends) to represent web —
  this is incorrect for turbine discs. Use layered modeling instead.
```

### 修复 4：强化 ProfileStation validator + 新增 anti-example

#### 4.1 强化 ProfileStation validator

**文件**：[models.py:12-33](file:///e:/auto_detection_process/integrations/engineering_tools/src/seekflow_engineering_tools/generative_cad/bases/axisymmetric/models.py#L12-L33)

**修改内容**：将 validator 从 `>` 改为 `>=`，与 A001 保持一致

```python
class ProfileStation(BaseModel):
    # ...
    z_front_mm: float = Field(
        description="该段起始 Z 位置（mm）。必须严格 < z_rear_mm（不允许零宽度段）。"
    )
    z_rear_mm: float = Field(
        description="该段结束 Z 位置（mm）。必须严格 > z_front_mm（不允许零宽度段）。"
    )

    @model_validator(mode="after")
    def validate_z(self):
        if self.z_front_mm >= self.z_rear_mm:
            raise ValueError("z_front_mm must be strictly < z_rear_mm (zero-width stations forbidden)")
        return self
```

**why**：在 schema 层面就拦截零宽度站点，与 A001 保持一致，而不是等到 preflight 阶段。

## 五、五层防线架构

修复完成后，系统建立了完整的五层防线，确保 LLM 使用正确的分层建模方式：

```
┌─────────────────────────────────────────────────────────────────┐
│ Layer 1: Schema 层（拦截）                                       │
│  └── ProfileStation.validate_z: z_front >= z_rear → 报错        │
│      拦截零宽度站点                                              │
├─────────────────────────────────────────────────────────────────┤
│ Layer 2: Prompt 层（引导）                                       │
│  └── NODE_PARAMS_SYSTEM_PROMPT: axisymmetric revolve_profile    │
│      建模规则（外轮廓单值、z_rear 严格大于、分层建模）           │
├─────────────────────────────────────────────────────────────────┤
│ Layer 3: Example 层（示范）                                      │
│  └── TURBINE_DISC_REFERENCE_EXAMPLE: 分层建模示例               │
│      revolve 纯外轮廓 + cut_annular_groove 创建 Web 凹陷        │
├─────────────────────────────────────────────────────────────────┤
│ Layer 4: Anti-example 层（禁止）                                 │
│  └── hourglass_shape_bad: 明确禁止沙漏形 profile                │
│      + regional_profile_stations_bad: 禁止区域描述              │
├─────────────────────────────────────────────────────────────────┤
│ Layer 5: Handler 层（能力）                                      │
│  └── cut_annular_groove: z_position_mm 支持精确定位 Z 区间      │
│      使分层建模成为可行方案                                      │
└─────────────────────────────────────────────────────────────────┘
```

### 防线协同

1. **Schema 层**（修复 4.1）：在 schema 层面拦截零宽度站点，快速失败
2. **Prompt 层**（修复 3）：明确告知 LLM axisymmetric 建模规则
3. **Example 层**（修复 2）：展示正确的分层建模，LLM 可模仿
4. **Anti-example 层**（修复 4.2）：明确禁止沙漏形 profile，避免 LLM 重蹈覆辙
5. **Handler 层**（修复 1）：提供 `z_position_mm` 能力，使分层建模可行

## 七、设计决策记录

### 决策 1：cut_annular_groove 扩展方式

**选择**：增加可选 `z_position_mm` 参数（而非新增 side_pocket 操作）

**理由**：
- 最小化修改，不新增 op，不需要注册新 handler/schema
- 向后兼容——缺失时回退到当前行为
- side 参数仍然决定切削方向，z_position_mm 只影响起始位置
- 已有 cut_annular_groove 的语义自然扩展

### 决策 2：TURBINE_DISC_REFERENCE_EXAMPLE 修正方式

**选择**：改为分层建模（而非简单圆柱）

**理由**：
- 展示正确的分层建模思路，LLM 可模仿
- 保留涡轮盘的复杂特征（Web 凹陷、bore、fir-tree slots）
- 与 NODE_PARAMS_SYSTEM_PROMPT 的规则一致

### 决策 3：ProfileStation validator

**选择**：从 `>` 改为 `>=`，与 A001 保持一致

**理由**：
- 在 schema 层面就拦截零宽度站点，快速失败
- 与 A001 保持一致，避免 validator 与 preflight 不一致
- 不影响现有合法输出（零宽度站点已被 A001 拦截）

### 决策 4：A001 不放宽

**选择**：保持 `zr > zf`（严格大于），不放宽

**理由**：
- 放宽 A001 不能解决根本问题——LLM 的建模方式仍然是错误的
- 应从 Prompt 层面引导 LLM 使用正确的分层建模方式
- sort-by-Z 算法虽然能处理零宽度站点，但产生的几何不是真正的涡轮盘

### 决策 5：side 参数语义

**选择**：保持 side 决定切削方向（front=+z, rear=-z），z_position_mm 只影响起始位置

**理由**：
- side 参数的语义清晰——决定切削方向
- z_position_mm 只影响起始位置，不改变方向
- 向后兼容——现有 side+depth 行为不变
