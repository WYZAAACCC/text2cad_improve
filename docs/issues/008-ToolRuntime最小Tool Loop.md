## 背景

虽然本项目不是完整 Agent 框架，但为了验证工具调用的端到端可靠性，必须提供一个最小运行时来串联所有模块。`ToolRuntime` 不叫 `Agent`，它只做最基本的 tool calling loop：发送消息 → 收到 tool_calls → 执行工具 → 写回结果 → 循环。

## 任务

### runtime.py — ToolRuntime

```python
class ToolRuntime:
    def __init__(
        self,
        *,
        tools: list[Any] | None = None,
        mcp_servers: list[Any] | None = None,
        api_key: str | None = None,
        base_url: str = "https://api.deepseek.com",
        strict: bool = False,
        strict_fallback: bool = True,
        repair: bool = True,
        trace: bool = True,
        max_steps: int = 6,
        max_result_chars: int = 12000,
    )

    def chat(
        self,
        *,
        model: str,
        messages: list[dict],
        **kwargs,
    ) -> ToolRuntimeResult
```

### Tool Loop 逻辑

```
1. 初始化 ToolRegistry，注册本地 tools
2. 生成 tools_schema（strict 模式则用 strict=True）
3. strict=True 时调用 check_strict_compatibility()
   - 失败 + fallback → 降级为非 strict，记录 trace 事件
   - 失败 + 不 fallback → 抛出 StrictSchemaError
4. for step in range(max_steps):
   a. 调用 DeepSeekClient.chat(model, working_messages, tools_schema)
   b. 记录 model_request 和 model_response 到 trace
   c. 没有 tool_calls → 返回 ToolRuntimeResult
   d. 有 tool_calls → 构造 assistant message 加入 working_messages
   e. 对每个 tool_call 调用 ToolExecutor.execute()
   f. 记录 tool_call_start/result/error 到 trace
   g. 构造 tool result message 加入 working_messages
5. 达到 max_steps → 返回 ToolRuntimeResult(final="...max_steps reached...")
```

### 严格的行为约束

- 不要扩展成完整 Agent 框架
- 不要添加记忆管理
- 不要添加权限控制
- 不要添加流式处理（第一版）
- 返回的 `ToolRuntimeResult.messages` 包含完整的对话历史
- 空工具列表时退化为普通聊天（不报错）

## 验收标准

- [ ] `ToolRuntime(tools=[add]).chat(model="deepseek-chat", messages=[...])` 端到端工作
- [ ] 模型选择调用工具 → 工具执行 → 模型收到结果 → 返回最终回答
- [ ] `strict=True` + schema 不兼容 + `strict_fallback=True` → 自动降级，trace 中记录 `strict_fallback`
- [ ] `strict=True` + schema 不兼容 + `strict_fallback=False` → 抛出 `StrictSchemaError`
- [ ] `max_steps` 耗尽时返回 `final="ToolRuntime stopped because max_steps was reached."`
- [ ] `trace=True` 时 `result.trace` 包含完整执行记录
- [ ] `result.trace.save("trace.json")` 写入合法 JSON
- [ ] 空工具列表时正常聊天
