## 背景

DeepSeek 工具调用过程涉及多个环节（请求、响应、修复、执行、错误），开发者调试时需要完整的执行记录。本 Issue 提供轻量级 trace 系统，记录所有事件的时间线，支持 JSON 导出。

## 任务

### events.py — 事件定义

```python
class TraceEvent(BaseModel):
    type: str
    timestamp: float  # time.time()
    data: dict[str, Any] = {}
```

14 种事件类型：

| 事件 | 说明 |
|------|------|
| `runtime_start` | Runtime 开始 |
| `strict_check` | Strict 检查结果 |
| `strict_fallback` | Strict 降级事件 |
| `model_request` | 发送模型请求 |
| `model_response` | 收到模型响应 |
| `tool_call_raw` | 收到原始 tool call |
| `json_repair` | JSON 修复执行 |
| `argument_coercion` | 参数类型纠正 |
| `tool_call_start` | 工具开始执行 |

| `mcp_tool_call_result` | MCP 工具结果 |
| `runtime_final` | Runtime 完成 |
| `runtime_error` | Runtime 错误 |

### recorder.py — 记录器

```python
class TraceRecord(BaseModel):
    trace_id: str  # uuid4
    started_at: str  # ISO datetime
    ended_at: str | None = None
    model: str | None = None
    events: list[TraceEvent] = []
    metadata: dict[str, Any] = {}

class TraceRecorder:
    def __init__(self, enabled: bool = True)
    def record(self, event_type: str, data: dict[str, Any] | None = None) -> None
    def finish(self) -> None  # 设置 ended_at
    def to_dict(self) -> dict
    def to_json(self) -> str
    def save(self, path: str) -> None
```

## 验收标准

- [ ] `TraceRecorder().record("tool_call_start", {"name": "add"})` 记录事件含时间戳
- [ ] `finish()` 后 `ended_at` 被设置且晚于 `started_at`
- [ ] `to_json()` 返回合法 JSON 字符串
- [ ] `save("trace.json")` 写入文件，内容可被 `json.load()` 解析
- [ ] `trace_id` 为非空 UUID
- [ ] `enabled=False` 时 `record()` 不产生事件
