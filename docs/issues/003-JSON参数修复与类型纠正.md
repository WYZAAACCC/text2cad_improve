## 背景

模型生成的 tool arguments 常常不是合法 JSON：单引号、尾随逗号、包裹在 markdown code block 中、使用 Python 字面量（`True`/`False`/`None`）、甚至在 JSON 前后附加解释性文本。参数类型也可能不匹配（字符串 `"123"` 应为整数）。本 Issue 实现自动修复，降低因格式问题导致的工具调用失败。

## 任务

### json_repair.py — 7 条修复规则

接口：

```python
class JsonRepairResult(BaseModel):
    ok: bool
    value: dict | None = None
    original: str
    repaired: str | None = None
    error: str | None = None
    applied_rules: list[str] = []

def repair_json_arguments(raw: str) -> JsonRepairResult
```

规则（按优先级）：

1. 去掉 markdown code block（```` ```json ... ``` ````）
2. 提取第一个 `{...}` JSON 对象（忽略前后解释文本）
3. 单引号转双引号
4. 去掉尾随逗号
5. Python `True` / `False` / `None` 转 `true` / `false` / `null`
6. 去掉 JSON 前后的解释性文本
7. 尝试 `json.loads`，成功则返回

### coercion.py — 5 种类型纠正

接口：

```python
def coerce_arguments(args: dict, parameters_schema: dict) -> tuple[dict, list[str]]
# 返回: (corrected_args, change_log)
```

纠正规则：

1. `"123"` → `123`，如果 schema 要求 `integer`
2. `"3.14"` → `3.14`，如果 schema 要求 `number`
3. `"true"` / `"false"` → `True` / `False`，如果 schema 要求 `boolean`
4. 单值 → `[value]`，如果 schema 要求 `array` 且可安全转换
5. 嵌套字段递归纠正

## 验收标准

- [ ] `repair_json_arguments("{'city': '杭州'}")` 返回 `ok=true, value={"city": "杭州"}, applied_rules=["single_quotes_to_double"]`
- [ ] `repair_json_arguments('{"city": "杭州",}')` 返回 `ok=true, applied_rules=["remove_trailing_commas"]`
- [ ] markdown code block 包裹的 JSON 被正确提取
- [ ] `Python True/False/None` 被转为 `true/false/null`
- [ ] 所有修复规则无法恢复时，返回 `ok=false` 并保留原始输入
- [ ] `coerce_arguments({"a": "12"}, {"properties": {"a": {"type": "integer"}}})` 返回 `({"a": 12}, ["coerced a: str -> int"])`
- [ ] `"true"` 转 `True`（boolean schema），`"3.14"` 转 `3.14`（number schema）
- [ ] 单值 `"hello"` 转 `["hello"]`（array schema 且可安全转换）
