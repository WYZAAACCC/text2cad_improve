## 背景

项目需要与 DeepSeek API 通信以发送 chat completion 请求。基于 `openai` Python SDK 做轻量封装，统一处理认证、工具调用解析和响应格式化。不做流式、不做重试策略（第一版）。

## 任务

### client.py — DeepSeekClient

```python
class DeepSeekClient:
    def __init__(
        self,
        api_key: str | None = None,
        base_url: str = "https://api.deepseek.com",
    )

    def chat(
        self,
        *,
        model: str,
        messages: list[dict],
        tools: list[dict] | None = None,
        tool_choice: str | dict | None = None,
        stream: bool = False,
        **kwargs,
    ) -> ChatResponse
```

### 关键行为

- 基于 `openai.OpenAI` 客户端，`base_url` 指向 DeepSeek API
- `api_key` 优先级：参数传入 > 环境变量 `DEEPSEEK_API_KEY`
- `base_url` 优先级：参数传入 > 环境变量 `DEEPSEEK_BASE_URL` > 默认 `https://api.deepseek.com`
- `model` 默认取 `DEEPSEEK_MODEL` 环境变量
- `chat()` 内部调用 `client.chat.completions.create()`，解析返回的 `ChatResponse`
- 支持从 response 中提取 `content`、`reasoning_content`、`tool_calls`、`finish_reason`、`usage`
- 支持 `extra_headers` 等透传参数
- 未设置 API key 时给出清晰错误提示

### ChatResponse 解析

- `content`：从 `choice.message.content` 提取
- `reasoning_content`：从 `choice.message.reasoning_content` 提取（DeepSeek R1 模型）
- `tool_calls`：从 `choice.message.tool_calls` 提取并转为 `list[ToolCall]`
- `usage`：从 `response.usage` 提取
- `raw`：保留完整原始响应对象

## 验收标准

- [ ] `DeepSeekClient()` 使用 `DEEPSEEK_API_KEY` 环境变量初始化
- [ ] `DeepSeekClient(api_key="sk-xxx")` 参数优先级高于环境变量
- [ ] `chat(model="deepseek-chat", messages=[{"role": "user", "content": "hi"}])` 返回 `ChatResponse`，`content` 非空
- [ ] 含 tool_calls 的响应正确解析为 `list[ToolCall]`
- [ ] `reasoning_content` 正确提取（使用 R1 模型时）
- [ ] `usage` 含 `prompt_tokens` / `completion_tokens` / `total_tokens`
- [ ] 未设置 API key 时给出可读的错误提示
