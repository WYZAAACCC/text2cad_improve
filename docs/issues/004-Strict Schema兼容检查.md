## 背景

DeepSeek 的 strict 模式对 tools schema 有更严格要求。如果 schema 不规范（如 anyOf/oneOf/allOf、过深嵌套、空 enum 等），API 直接报错，开发者排查困难。本 Issue 提供一个前置检查器，在请求发送前验证 schema 兼容性，并支持自动降级。

## 任务

### strict.py — 兼容检查器

```python
class StrictCheckIssue(BaseModel):
    level: str  # "warning" | "error"
    path: str  # JSON path 指向问题字段
    message: str
    suggestion: str | None = None

class StrictCheckResult(BaseModel):
    ok: bool
    issues: list[StrictCheckIssue] = []

def check_strict_compatibility(tools_schema: list[dict]) -> StrictCheckResult
```

### 10 项检查

| # | 检查项 | 级别 |
|---|-------|------|
| 1 | `function.name` 是否合法（字母/数字/下划线/连字符） | error |
| 2 | `description` 是否为空 | warning |
| 3 | `parameters` 是否为 `object` 类型 | error |
| 4 | `properties` 是否存在（非空 object） | error |
| 5 | `required` 字段是否与 properties 一致 | warning |
| 6 | 是否存在 `anyOf` / `oneOf` / `allOf` | warning |
| 7 | 是否存在过深嵌套（> 3 层） | warning |
| 8 | `enum` 是否为空数组 | error |
| 9 | `default` 是否可能导致 strict 不兼容 | warning |
| 10 | 是否存在 strict 不支持的其他复杂 schema 特性 | warning |

### 集成点（预留给 Issue 8）

`ToolRuntime` 在初始化时：
- `strict=True` 时调用 `check_strict_compatibility()`
- 检查失败 + `strict_fallback=True` → 降级为非 strict，记录 `strict_fallback` trace 事件
- 检查失败 + `strict_fallback=False` → 抛出 `StrictSchemaError`

## 验收标准

- [ ] 合法 schema 返回 `ok=true, issues=[]`
- [ ] 缺少 `parameters` 的 schema 返回 error 级别 issue
- [ ] `parameters.type != "object"` 返回 error 级别 issue
- [ ] 含 `anyOf` / `oneOf` / `allOf` 返回 warning 级别 issue
- [ ] 空 `description` 返回 warning 级别 issue
- [ ] 空 `enum` 返回 error 级别 issue
- [ ] 每个 issue 包含清晰的 `path` 和 `message`
- [ ] `suggestion` 字段为有问题的 issue 提供修复建议
