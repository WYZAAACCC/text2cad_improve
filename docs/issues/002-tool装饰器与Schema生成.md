## 背景

DeepSeek API 不认识 Python 函数，需要 JSON 格式的工具描述。开发者手写 tools schema 容易出错且与代码不同步。本 Issue 提供一个 `@tool` 装饰器，自动从 Python 函数签名生成 DeepSeek-compatible tools schema。

## 任务

### decorator.py — `@tool` 装饰器

- 支持 `@tool` 无参数形式（自动从函数名和 docstring 提取 name/description）
- 支持 `@tool(name=..., description=...)` 有参数形式
- 被装饰的函数保留 `__name__`、`__doc__` 等元信息
- 装饰器返回 `ToolDefinition`

### schema.py — 类型映射

支持 9 种 Python 类型 → JSON Schema：

| Python 类型 | JSON Schema 类型 |
|---|---|
| `str` | `string` |
| `int` | `integer` |
| `float` | `number` |
| `bool` | `boolean` |
| `list[str]` / `list[int]` | `array`（含 items） |
| `dict` | `object` |
| `Literal["a", "b"]` | `enum` |
| `Optional[str]` | 不进入 required |
| Pydantic Model | object schema（含 properties 和 required） |

- 从 docstring 提取 `description`
- 从函数签名的 `inspect.Parameter` 提取参数名、类型、默认值
- 无默认值的参数进入 `required` 列表

### registry.py — ToolRegistry

```python
class ToolRegistry:
    def register(self, tool: Callable | ToolDefinition) -> ToolDefinition
    def get(self, name: str) -> ToolDefinition
    def has(self, name: str) -> bool
    def list(self) -> list[ToolDefinition]
    def to_deepseek_tools(self, strict: bool = False) -> list[dict]
```

- 注册时自动对 `Callable` 调用 `@tool` 装饰器
- 重名工具注册时抛出 `ToolSchemaError`
- `to_deepseek_tools()` 导出 DeepSeek API 格式的 `[{"type": "function", "function": {...}}]`
