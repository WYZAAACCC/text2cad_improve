## 背景

MCP（Model Context Protocol）是 AI 工具调用的通用协议，已有大量 MCP server（文件系统、数据库、GitHub 等）。本 Issue 让 DeepSeek 能通过 stdio 传输连接 MCP server，自动发现工具并执行。第一版只做 stdio，不做 SSE / HTTP。

## 任务

### config.py — 配置

```python
class MCPServerConfig(BaseModel):
    name: str
    transport: str = "stdio"
    command: str
    args: list[str] = []
    env: dict[str, str] = {}

    @classmethod
    def stdio(cls, name: str, command: str, args: list[str] | None = None, env: dict[str, str] | None = None) -> "MCPServerConfig"
```

### client.py — MCP 连接管理

- 使用 `mcp` 包的 stdio client（`mcp>=1.0.0` 作为可选依赖）
- 通过 subprocess 启动 MCP server
- 管理 session 生命周期（启动、心跳、关闭）
- 拉取 MCP server 提供的 tools 列表

### adapter.py — 工具转换

```python
def mcp_tool_to_deepseek_tool(server_name: str, mcp_tool: Any, strict: bool = False) -> dict
```

- 将 MCP tool 的 inputSchema 转为 DeepSeek tools schema
- 工具名加 namespace：`{server_name}.{tool_name}`
- 处理 MCP schema 到 JSON Schema 的兼容差异

### executor.py — MCP 执行器

```python
class MCPToolExecutor:
    async def execute(self, tool_call: ToolCall) -> ToolExecutionResult
    def execute_sync(self, tool_call: ToolCall) -> ToolExecutionResult  # 同步包装
```

- 解析工具名 → 提取 server namespace → 找到对应 MCP session → 调用 `call_tool`
- 标准化返回结果为 `ToolExecutionResult`
- 标准化错误格式

### 安全

- README 必须包含安全提示：只连接可信的 MCP server，它们可能访问本地文件、网络或系统资源

### example

创建 `examples/04_mcp_filesystem.py`：连接 `@modelcontextprotocol/server-filesystem`，完成基本文件操作
