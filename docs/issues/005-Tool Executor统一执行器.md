## 背景

工具执行涉及多个步骤：参数解析、JSON 修复、类型纠正、函数调用、异常处理、超时控制、结果序列化和截断。如果这些逻辑分散在各处，代码会充满重复的错误处理。本 Issue 提供一个统一的 `ToolExecutor`，处理所有执行路径和异常情况。

## 任务

### executor.py — 统一执行

```python
class ToolExecutor:
    def __init__(
        self,
        registry: ToolRegistry,
        repair: bool = True,
        max_result_chars: int = 12000,
    )

    def execute(self, tool_call: ToolCall) -> ToolExecutionResult
```

### 执行流程

```
ToolCall → 查找工具（registry）
         → arguments 是 str？→ JSON parse → 失败则 repair → 失败则返回 error
         → coercion（如果 repair=True）
         → 调用 func(**args)
         → 捕获异常 → 返回 ToolExecutionResult(ok=false)
         → 序列化 result → 不可序列化则 str()
         → 截断（超过 max_result_chars）
         → 返回 ToolExecutionResult(ok=true)
```

### 必须处理的 9 种情况

1. 工具名不存在 → `ToolExecutionResult(ok=false, error="Tool not found: xxx")`
2. arguments 是字符串 → JSON parse
3. JSON 错误 → repair 后 retry
4. 参数缺失 → 记录 error
5. 参数类型错误 → coercion
6. 函数执行异常 → 捕获，返回 `ok=false`
7. 执行超时 → 不抛异常，返回 `ok=false, error="Tool execution timed out"`
8. 返回结果不可 JSON 序列化 → `str()` 转换
9. 返回结果过长 → 截断并标注 `[Tool result truncated: original length X chars, kept Y chars]`

## 验收标准

- [ ] `execute(ToolCall(name="add", arguments={"a": 1, "b": 2}))` 返回 `ok=true, result=3`
- [ ] 工具不存在时返回 `ok=false`，不抛异常
- [ ] arguments 为 JSON 字符串 `'{"a": 1, "b": 2}'` 时正确解析
- [ ] arguments 为单引号 JSON `"{'a': 1}"` 时自动 repair 后执行
- [ ] 工具内部抛出异常时返回 `ok=false, error=...`，不中断执行器
- [ ] 返回结果超过 `max_result_chars` 时截断并标注
- [ ] 返回结果不可 JSON 序列化时降级为 `str()` 或返回错误信息
- [ ] `repair=False` 时跳过 repair 和 coercion 步骤
