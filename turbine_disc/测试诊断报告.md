## 6. 问题 2: Slot Profile 深度递减导致校验失败

### 6.1 现象

```json
// validation_report.json
{
  "ok": false,
  "stage": "params",
  "issues": [{
    "code": "invalid_params",
    "message": "slot profile station depths must be nondecreasing",
    "node_id": "n_rim_slots"
  }],
  "stages_run": ["structure", "root_terminal", "registry", "params"]
}
```

校验在 **第4个阶段 (params)** 失败，未能进入后续 10 个阶段。

### 6.2 根因

**测试 Prompt 中的描述（我写的）：**
```
第1站（最深处）：depth=12mm，half_width=4mm
第2站：depth=8mm，half_width=5.5mm  
第3站（最浅处）：depth=4mm，half_width=7mm
```

**LLM 按 Prompt 顺序输出:**
```json
// llm_raw.json → n_rim_slots.params.slot_profile.stations
"stations": [
  {"depth_mm": 12, "half_width_mm": 4},   // 最深 → 第1站
  {"depth_mm": 8,  "half_width_mm": 5.5}, // 第2站
  {"depth_mm": 4,  "half_width_mm": 7}    // 最浅 → 第3站  ← 递减!
]
```

**系统约束 (CutRimSlotPatternParams → RimSlotProfile → SlotProfileStation):**

`SlotProfileStation` 的 validator:
```
depths must be nondecreasing
```

即 `stations[i+1].depth_mm >= stations[i].depth_mm`

**语义含义:** `depth_mm` 表示从轮缘表面向内的深度。第1站是最浅的（靠近外表面），最后一站是最深的（最靠近轮盘中心）。站点越深，宽度越窄（模拟 fir-tree 从外到内逐渐变窄的几何形状）。

**但是**，Prompt 使用了中文的习惯表述"第1站最深→第3站最浅"（从内向外编号），而系统期望的编号方向相反（从外向内，深度递增）。

### 6.3 LLM 为何不自行纠正

这是 Prompt 设计缺陷，而非 LLM 错误。LLM 忠实遵循了 Prompt 中的站点顺序：

1. LLM 接收到 Prompt: "第1站 depth=12 → 第3站 depth=4"
2. 系统要求站点的 depth 必须非递减 (shallow → deep)
3. LLM 没有足够上下文来理解系统内部对站点排序的要求
4. 即使 LLM 收到 `tool_schema_compiler` 编译的 JSON Schema，DeepSeek `strict=False` 模式可能未严格校验此约束

### 6.4 自动修复为何未处理

查看 `autofix_report.json`，autofixer 的 20+ 规则中不包含 slot profile station 排序规则。autofixer 应用的 3 条规则是：
1. `fix_chamfer_fillet_optional` — chamfer 降级为 optional
2. `fix_phase_ordering` — 修正 phase 排序
3. `fix_null_hints` — null → {}

**autofixer 缺失的规则:** `fix_slot_station_order` — 对 slot_profile.stations 按 depth_mm 排序

### 6.5 修复

```python
# 手动重排: 按 depth_mm 升序
stations.sort(key=lambda s: s['depth_mm'])
# 结果: [4→7, 8→5.5, 12→4]
```

### 6.6 预防

1. **Prompt 层面:** 明确要求站点从浅到深排列
2. **autofixer 层面:** 添加 `fix_slot_station_order` 规则
3. **L2 Prompt 层面:** 在 `LEVEL2_AUTHORING_SYSTEM_PROMPT` 中添加 slot ordering 指令
4. **schema 层面:** 考虑在 error message 中提示正确的排列方向

**涉及文件:**
- `integrations/.../generative_cad/dialects/axisymmetric/params.py` — CutRimSlotPatternParams 定义
- `integrations/.../generative_cad/authoring/auto_fixer.py` — 修复规则列表 (缺少 slot order 规则)

## 7. 问题 3: Bore 内壁共面切割导致 2-Solid 分裂

### 7.1 现象

```
Runtime error: geometry postcheck failed: 
Final geometry has 2 solid bodies (expected 1).
This means boolean_union failed to merge components.
Check for grazing contact or non-intersecting solids in the assembly.
```

调试过程：

| 测试 | 节点 | 结果 |
|---|---|---|
| Step 0 | `n_revolve` only | ✅ 1 Solid, 20 KB |
| Step 0b | `n_revolve` + `n_center_bore` | ❌ 2 Solids! |

**确认:** `cut_center_bore` 是问题根因。

### 7.2 根因

**LLM 生成的 revolve_profile 截面:**

```
bore_bottom: r=60mm, z=5→6     ← 内壁底部 (1mm 薄片)
bore_wall:   r=60mm, z=6→115   ← 内壁竖直面
hub_front:   r=130mm, z=5→42.5
...
```

`revolve_profile` 创建了一个**内径恰好为 r=60mm 的中空回转体**。内壁表面在半径 60mm 处。

**cut_center_bore 操作:**
```json
{"diameter_mm": 120, "through_all": true}  // 半径 = 60mm
```

`cut_center_bore` 创建一个直径 120mm (半径 60mm) 的圆柱体，然后执行 **boolean cut** (减法操作)。

### 7.3 3D 几何层面发生了什么

```
截面示意图 (R-Z 平面):

R(mm)
250 ┤                              ┌─────┐  Rim
    │                              │     │
210 ┤                        ┌─────┘     │  Web 外缘
    │                        │  Web      │
130 ┤         ┌──────────────┘           │  Hub 外缘
    │         │  Hub                     │
 60 ┤  ┌──────┘                          │  ← 内壁表面 (与 bore cut 共面!)
    │  │                                  │
  1 ┤  │ ←── 修复后: 中心线填充到 r=1     │
    │  │                                  │
  0 ┼──┴──────────────────────────────────┤  ← Z 轴
    0    20    42.5    77.5   100   115  120
         ← 前 →              ← 后 →

图例:
  ┌─ = revolve_profile 生成的实心材料
  ── = 内壁/外壁表面
  ←─ = bore cut 圆柱体 (r=60mm) 的切割位置
```

**共面切割问题:**

1. `revolve_profile` 生成的 solid 在 r=60mm 处有一个**内表面**
2. `cut_center_bore` 的切割圆柱体也是 r=60mm
3. OCCT 几何内核执行 boolean cut 时，切割体与被切体的**内表面完全重合**
4. 这种共面情况可能导致：
   - 切割体实际上没有穿透 solid（只在表面上"滑过"）
   - 或者产生非流形 (non-manifold) 的结果
   - CAD 内核将结果拆分为 2 个独立 solid body

**类比:** 这就像用一个恰好和杯子内壁一样大的钻头去钻孔——如果钻头和内壁刚好贴合，钻孔操作不会去除任何材料，甚至可能把杯子切成两半。

### 7.4 为什么验证层没有提前检测

1. **preflight_component** 检查了 bore 半径 vs profile_max_radius 的基本范围，但没有检测共面情况
2. 共面检测需要精确的几何分析，通常在运行时才会暴露
3. OCP/BRepCheck 的 BRep 检查也未能捕获共面导致的拓扑分裂

### 7.5 修复

```python
# 修复前: 内壁刚好在 r=60mm, bore cut 也是 r=60mm
# 修复后: 从 r=1.0 开始填充中心, bore cut 可以切除实际材料

node['params']['profile_stations'] = [
    {'r_mm': 1.0, 'z_front_mm': 0, 'z_rear_mm': 120, 'label': 'centerline_fill'},
    {'r_mm': 130, 'z_front_mm': 5, 'z_rear_mm': 115, 'label': 'hub_body'},
    {'r_mm': 210, 'z_front_mm': 42.5, 'z_rear_mm': 77.5, 'label': 'web'},
    {'r_mm': 250, 'z_front_mm': 20, 'z_rear_mm': 100, 'label': 'rim'},
]
```

**关键变化:**
1. 添加 `r=1.0` 中心线填充站（从 Z=0 到 Z=120）
2. 不再需要 bore_bottom 和 bore_wall 两个站点
3. `cut_center_bore` 现在从实心材料中切除中心，不存在共面问题

**注意:** `r_mm` 必须 > 0（`DimExprOrFloat` 拒绝 0 和负数），因此使用 `r=1.0` 作为最小填充半径。

### 7.6 预防

1. **preflight_component 中添加共面检测:** 当 bore 半径等于 profile 内壁半径时发出警告
2. **auto_fixer 规则:** 检测 revolve_profile + cut_center_bore 组合，自动添加 centerline_fill
3. **L2 Prompt 建议:** 在 `LEVEL2_AUTHORING_SYSTEM_PROMPT` 或 base_package 中说明"不要在 profile 中定义内壁，应让 revolve 创建实心圆柱，使用 cut_center_bore 创建中心孔"
4. **geometry_preflight 增强:** 检测 bore_cutter_radius >= profile_min_radius 的情况

**涉及文件:**
- `integrations/.../generative_cad/dialects/axisymmetric/handlers.py` — revolve_profile handler
- `integrations/.../generative_cad/validation/geometry_preflight.py` — 几何预检
- `integrations/.../generative_cad/base_packages/axisymmetric/package.py` — LLM 说明文档

## 8. 问题 4: Chamfer 几何计算降级

### 8.1 现象

```
Warning: Optional feature 'apply_safe_chamfer' skipped on node 'n_chamfer': 
chamfer failed at distance=1.5mm and fallback distance=0.75mm.
Part is valid without this feature.
```

### 8.2 根因

1. `apply_safe_chamfer` 尝试对 `target="all_external_edges"` 应用 1.5mm 倒角
2. 由于前一步的 `n_rim_slots` 在轮缘上创建了 60 个复杂榫槽
3. 每个榫槽引入了大量新边（约 8-12 条边/槽 × 60 = ~600 条新边）
4. 加上榫槽轮廓中的尖锐转角和狭窄通道 (half_width 最小 4mm)
5. OCCT 的 chamfer 算法在这种情况下通常失败：
   - 某些边的几何空间不足以容纳 1.5mm chamfer
   - 相邻边的 chamfer 可能相互干涉
   - 榫槽根部的凹角 chamfer 会自交

### 8.3 处理机制

`apply_safe_chamfer` 内置降级策略：
```
attempt chamfer(1.5mm) → 失败
    ↓
fallback chamfer(0.75mm) → 也失败
    ↓
因为 auto_fixer 已将 required=false, degradation_policy="may_skip_with_warning"
    ↓
记录 warning + degraded_feature, 跳过此特征, 零件仍然有效
```

### 8.4 影响

- 无功能影响 — 涡轮盘几何体完整 (1 Solid, 封闭实体)
- 轮缘锐边保留原状

## 10. 改进建议

### 10.1 高优先级 (会导致管线失败)

#### A. 在 auto_fixer 中添加 slot station 排序规则

```python
# authoring/auto_fixer.py 新增
def _fix_slot_station_order(doc: dict) -> bool:
    """Sort slot profile stations by depth_mm (nondecreasing)."""
    changed = False
    for node in doc.get("nodes", []):
        sp = node.get("params", {}).get("slot_profile", {})
        stations = sp.get("stations", [])
        if stations and stations != sorted(stations, key=lambda s: s.get("depth_mm", 0)):
            sp["stations"] = sorted(stations, key=lambda s: s.get("depth_mm", 0))
            changed = True
    return changed
```

#### B. 在 preflight 中添加 bore-vs-profile 共面检测

```python
# validation/geometry_preflight.py 新增
def _check_bore_profile_coplanarity(component, nodes, registry):
    """Check if bore diameter equals profile inner radius (coplanar cut)."""
    for node in nodes:
        if node.op == "cut_center_bore":
            bore_r = node.params.get("diameter_mm", 0) / 2
        if node.op == "revolve_profile":
            profile_min_r = min(s.r_mm for s in node.params.profile_stations)
    if abs(bore_r - profile_min_r) < 0.01:
        yield ValidationIssue(
            code="bore_profile_coplanar",
            severity="error",
            message=f"Bore radius ({bore_r}mm) equals profile inner radius ({profile_min_r}mm). "
                    "This causes coplanar boolean cut. Add centerline fill to profile."
        )
```
