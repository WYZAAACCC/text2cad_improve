## 三、当前面临的问题

### 问题 #1 (P1): fillet_sketch 顶点提取不稳定

**现象**: 最新测试中，`fillet_sketch` 对 disc 和 cutter 都失败：
```
WARN: vertex extraction failed (BRep_API: command not done)
WARN: vertex extraction failed (BRepAdaptor_Curve::No geometry)
```

**原因**: 从 CadQuery wire 提取顶点坐标的 `wire.Vertices()` 方法对某些几何形状（特别是 mirror_profile 处理后的对称轮廓）返回不可用的几何数据。

**当前行为**: fillet gracefully degrades（保留原始无圆角的 profile），不阻塞管线。

**建议修复方向**:
- 方案 A: 修复顶点提取逻辑，使用 `TopExp_Explorer` + `BRep_Tool::Pnt` 直接获取顶点坐标
- 方案 B: 在 `handle_add_polyline` 中持久化原始点坐标，绕过 wire 顶点提取
- 方案 C: 使用 OCCT `BRepFilletAPI_MakeFillet2d` 直接在 face 上操作

### 问题 #2 (P2): LLM 生成的 fir-tree 方向与用户期望相反

**现象**: 用户期望 "外宽里窄"（mouth 宽 → root 窄），但 LLM 生成的 prompt 示例仍是 "窄颈—宽齿肩—窄底部"（mouth 窄 → lobe 宽）。

**原因**: `prompts.py` rule 30c 中的 fir-tree 示例 polyline 方向未更新。

**建议修复**: 更新 prompts.py 中的 fir-tree 示例点坐标，使 mouth Y 坐标更大。

### 问题 #3 (P2): LLM 有时使用 mirror_profile + place_component

**现象**: 部分测试中 LLM 使用 `mirror_profile` 创建对称 cutter 轮廓，或使用 `place_component` 预定位 cutter。

**原因**: prompt 规则不够强，LLM 有时忽略规则。

**当前缓解**: `handle_circular_pattern_component` 有双重定位检测，`handle_close_profile` 对已闭合 wire 容错。

### 问题 #4 (P3): fillet_sketch 产生过多弧线点导致 BRepCheck 失败

**现象**: 当 fillet_sketch 成功执行时，生成的弧线点过多（100+ point），导致 CadQuery extrude 后 BRepCheck=False，最终 boolean_cut 失败。

**当前缓解**: 已将 `num_segments` 从 8 降为 2，将共线检测阈值从 0.999 提高到 0.95。

### 问题 #5 (P3): deepseek-v4-pro L1 routing 不稳定

**现象**: 在无 spatial 上下文时，L1 routing 经常 4 次尝试全部失败。

**当前缓解**: `server/main.py` 中当 `forceRoute` 设置时自动跳过 L1 routing。
